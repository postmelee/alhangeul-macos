# Task #154 수행계획서: 제품 identity를 Alhangeul로 통일

## 목적

첫 public release 전에 `AlhangeulMac`으로 남아 있는 앱 파일명, bundle id, Xcode project/product/executable, extension 산출물, Cask app stanza, 문서 표현을 `Alhangeul` 기준으로 통일한다. 배포 이후 identity 변경은 설치본, LaunchServices, PlugInKit, Homebrew, App Store 경로에 영향을 크게 주므로 v0.1 배포 전에 확정하는 것을 목표로 한다.

## 배경

Task #148 배포 수준 결정 과정에서 사용자-facing 이름뿐 아니라 `.app` 파일명, bundle id, Xcode 내부 이름, Cask app stanza까지 `Alhangeul`로 통일하고 싶다는 작업지시자 요구가 확인됐다. 기존 문서에는 ExtensionKit lookup 안정성을 이유로 `AlhangeulMac.app`과 `com.postmelee.alhangeulmac` 계열을 유지한다고 되어 있으나, public release 전 identity 확정 관점에서는 한 번에 rename하는 편이 장기적으로 더 명확하다.

초기 조사에서 `AlhangeulMac`/`alhangeulmac` 사용처는 다음 범위에 걸쳐 있었다.

- `project.yml`: project name, product name, executable name, bundle id
- `Sources/**/Info.plist`: 기본 display/name, UTI identifier, extension content type
- `Sources/**/Resources/*/InfoPlist.strings`: 영어/한국어 localized display name
- `Sources/HostApp/Services/*`, `Sources/ThumbnailExtension/*`: extension bundle id, appex 이름, domain, queue label, share directory
- `scripts/package-release.sh`, `scripts/release.sh`: project/app/DMG volume/staging 이름
- `Casks/alhangeul-macos.rb`: `app "AlhangeulMac.app"`
- `README.md`, `mydocs/manual/*`, `mydocs/tech/*`, `.github/*`: build/run/release/Finder smoke 안내와 정책 문서

## 범위

포함:

- `.app` filesystem bundle name: `AlhangeulMac.app`에서 `Alhangeul.app`으로 변경
- Xcode project/product/executable/appex 산출물 이름을 `Alhangeul` 계열로 변경
- bundle identifier를 `com.postmelee.alhangeul` 계열로 변경
- 앱 내부 domain, queue label, temporary/share directory 등 `alhangeulmac` 문자열 정리
- UTI identifier를 `com.postmelee.alhangeul.hwp`/`com.postmelee.alhangeul.hwpx` 계열로 변경할지 Stage 1에서 최종 확인하고, hard blocker가 없으면 함께 변경
- 영어 사용자 표시명은 `Alhangeul`, 한국어 사용자 표시명은 `알한글` 기준으로 정리
- Quick Look/Thumbnail extension 표시명과 appex 이름 정리
- Cask `app` stanza와 release/package script 산출물 경로 정리
- README, release guide, build/run guide, architecture 문서의 정책과 명령 예시 갱신
- 기존 `AlhangeulMac` 설치본/등록 잔여물 처리 기준 문서화
- Debug build, Release build, package/release script help, Finder/Quick Look smoke 검증 범위 정리

제외:

- 실제 public release 게시
- GitHub Release asset upload
- Homebrew tap push 또는 PR 생성
- App Store Connect 제출
- signing certificate, notarytool credential, secret 변경
- viewer 기능, renderer 기능, 문서 파싱 동작 변경

## 설계 방향

- public release 전 identity 확정이 목적이므로 rename 범위는 좁게 보수적으로 나누되, `AlhangeulMac`과 `com.postmelee.alhangeulmac`가 사용자/배포 경로에 남지 않도록 끝까지 점검한다.
- `project.yml`을 Xcode project 원본으로 유지하고, generated `AlhangeulMac.xcodeproj`는 직접 수정하지 않는다.
- bundle id 변경에 맞춰 extension lookup, `pluginkit` 확인 문자열, app 내부 extension status model을 함께 바꾼다.
- UTI identifier 변경은 HostApp exported type, Quick Look supported type, Thumbnail supported type, open panel allowed type이 모두 같은 값으로 움직이도록 처리한다.
- 기존 `com.hancom.*` 호환 UTI는 유지한다.
- release 산출물 파일명 `alhangeul-macos-<version>.dmg`와 Cask token `alhangeul-macos`는 저장소/배포 channel 이름으로 유지한다. 바꾸면 GitHub Release URL과 Homebrew token 영향이 커지므로 이번 identity rename의 주 대상에서 제외한다.
- 변경 후 Finder/LaunchServices/PlugInKit stale 등록 가능성을 문서에 명시하고 smoke test를 release package 기준으로 수행한다.

## 잠정 단계

### Stage 1: rename 대상과 호환성 경계 조사

- `AlhangeulMac`, `alhangeulmac`, `AlhangeulMac.app`, `AlhangeulMacHost`, `AlhangeulMacPreview`, `AlhangeulMacThumbnail` 사용처를 전체 목록화한다.
- product/project/bundle id/UTI/domain/documentation을 변경 필수, 유지, legacy 언급으로 분류한다.
- `alhangeul-macos` 저장소명, Cask token, release DMG 파일명은 유지할지 최종 확인한다.
- 산출물 이름 변경이 package/release script와 Finder smoke test에 미치는 영향을 정리한다.

### Stage 2: Xcode project identity와 bundle/UTI 정합화

- `project.yml`, Info.plist, localized strings를 변경한다.
- HostApp, Quick Look, Thumbnail bundle id와 UTI/content type이 같은 identity 계열을 가리키도록 맞춘다.
- `xcodegen generate` 후 Debug build를 통해 project/product rename compile 문제가 없는지 확인한다.

### Stage 3: Swift 코드와 script/Cask rename 반영

- extension status lookup, open panel UTI, scheme handler domain, queue label, temporary directory를 정리한다.
- `scripts/package-release.sh`, `scripts/release.sh`, Cask `app` stanza를 `Alhangeul.app` 기준으로 바꾼다.
- script syntax와 release help/fail-fast를 검증한다.

### Stage 4: 문서와 smoke test 기준 갱신

- README, build/run guide, release guide, architecture 문서의 `AlhangeulMac`/`alhangeulmac` 정책을 새 기준으로 갱신한다.
- 기존 설치본/등록 잔여물 처리 기준을 문서화한다.
- Debug/Release build, release package, Finder/Quick Look/Thumbnail smoke test 명령을 새 이름 기준으로 점검한다.

### Stage 5: 최종 검증과 보고서 정리

- 전체 stale string scan을 수행하고, 남은 `AlhangeulMac`/`alhangeulmac`가 legacy 문맥인지 확인한다.
- shell syntax, XcodeGen, build, package, Finder/extension smoke 가능 범위 검증을 정리한다.
- 단계 보고서와 최종 보고서에 release #148과의 후속 영향, Task #154 완료 기준을 남긴다.

## 검증 계획

- `rg -n "AlhangeulMac|alhangeulmac|AlhangeulMacHost|AlhangeulMacPreview|AlhangeulMacThumbnail|com\\.postmelee\\.alhangeulmac" ...`
- `xcodegen generate`
- `xcodebuild -project Alhangeul.xcodeproj -scheme HostApp -configuration Debug -derivedDataPath build.noindex/DerivedData CODE_SIGNING_ALLOWED=NO build`
- `bash -n scripts/package-release.sh scripts/release.sh`
- `./scripts/release.sh --help`
- `./scripts/package-release.sh 0.1.0`
- `./scripts/release.sh --skip-notarize 0.1.0` 가능 시 수행
- `pluginkit -mAvvv | grep com.postmelee.alhangeul`
- Finder 통합 smoke: `qlmanage -t -x` 기준으로 HWP/HWPX thumbnail/preview 확인
- `git diff --check`

실제 notarized public release, GitHub Release 게시, Homebrew tap 반영은 이 task의 검증 범위 밖이며 별도 승인 후 수행한다.

## 리스크

- bundle id와 UTI identifier를 바꾸면 기존 설치본과 LaunchServices/PlugInKit 등록 정보가 남아 두 앱/두 extension처럼 보일 수 있다.
- `AlhangeulMac.xcodeproj`에서 `Alhangeul.xcodeproj`로 project 이름을 바꾸면 문서와 CI/명령 예시를 빠짐없이 바꿔야 한다.
- executable/process 이름을 `Alhangeul`로 바꾸면 기존 smoke test의 `pgrep -x AlhangeulMacHost` 같은 검증이 깨진다.
- UTI identifier 변경은 exported/imported type과 Quick Look/Thumbnail supported type이 불일치하면 Finder 통합이 깨질 수 있다.
- `alhangeul-macos` 저장소명, release DMG 파일명, Homebrew Cask token까지 바꾸면 배포 URL 영향이 커지므로 이번 task에서는 유지하는 편이 안전하다.
- Task #148 배포 문서 작업과 겹치는 문서가 있으므로 merge 순서와 conflict에 주의해야 한다.

## 승인 요청 사항

이 수행계획서 기준으로 Task #154의 구현계획서를 작성해도 되는지 승인 요청한다.

승인 시 다음 전제를 적용한다.

- 최종 제품 identity 철자는 `Alhangeul`로 고정한다.
- `.app`, bundle id, Xcode project/product/executable, appex, Cask app stanza는 `Alhangeul` 계열로 변경하는 방향으로 진행한다.
- 저장소명, release DMG 파일명, Homebrew Cask token은 `alhangeul-macos`로 유지한다.
- 실제 public release 게시, notarization submission, Homebrew tap 반영은 수행하지 않는다.
