# Task M020 #65 수행계획서

## 목적

rhwp core가 생성하는 기준 렌더링 결과와 macOS native renderer 결과를 같은 입력 파일에서 반복 비교할 수 있는 디버깅 절차를 정리한다.

완료 후 작업자는 특정 HWP/HWPX 파일에 대해 render tree JSON, core SVG, native PNG, 비교용 산출물을 한 위치에 생성하고, 차이가 발생했을 때 원인을 core 구현 한계, render tree export 문제, Swift 디코딩/렌더링 문제, HostApp 표시 문제로 나누어 판단할 수 있어야 한다.

## 배경

`table-in-tbox.hwp`를 수동으로 확인한 결과, 현재 앱의 native renderer PNG와 rhwp core `render_page_svg_native` 기반 SVG/PNG 사이에 눈에 띄는 시각 차이가 있었다.

저장소에는 이미 `scripts/validate-stage3-render.sh`가 있어 native renderer smoke PNG를 만들 수 있고, `RhwpDocument.renderPageSVG(at:)`와 `rhwp_render_page_svg`를 통해 core SVG도 얻을 수 있다. 하지만 두 경로를 한 명령으로 비교하고, render tree JSON과 pixel diff까지 남기는 표준 디버깅 절차는 아직 없다.

## 범위

포함:

- 특정 HWP/HWPX 파일의 1페이지 이상을 대상으로 native renderer PNG를 생성하는 디버깅 흐름 정리
- 같은 입력에서 rhwp core SVG를 export하는 흐름 정리
- render tree JSON dump 산출물 추가 또는 이를 얻는 절차 정리
- 가능한 환경에서는 core SVG rasterize PNG와 native PNG의 pixel diff 산출
- 산출물 디렉터리와 파일명 규칙 정리
- 디버깅 문서 작성

제외:

- 실제 native renderer 품질 수정
- rhwp core 수정
- HostApp viewer UI 변경
- 한컴 viewer와의 자동 비교
- 모든 플랫폼에서 동일한 SVG rasterizer를 보장하는 배포용 도구 구성

## 설계 방향

기존 `RhwpDocument`, `RenderTree`, `CGTreeRenderer` 경로를 재사용한다. 제품 경로의 renderer를 우회하지 않고, 현재 HostApp/Quick Look/Thumbnail이 사용하는 render tree 기반 CoreGraphics 렌더링과 같은 해석 결과를 디버깅 산출물로 남긴다.

core 기준 결과는 `rhwp_render_page_svg`를 통해 얻는 `render_page_svg_native` SVG로 둔다. 이 SVG는 제품 fallback 기준이 아니라 진단 기준 산출물로만 취급한다.

pixel diff는 환경 의존성이 있으므로 필수 산출물과 선택 산출물을 분리한다. 필수 산출물은 render tree JSON, core SVG, native PNG이고, SVG rasterize PNG와 diff PNG는 macOS 기본 도구 또는 사용 가능한 로컬 도구로 생성 가능한 경우에만 만든다.

## 리스크

- macOS Quick Look 기반 SVG rasterize는 OS, 캐시, sandbox 상태에 따라 결과가 달라질 수 있다.
- 외부 rasterizer가 설치되어 있지 않은 환경에서는 pixel diff를 자동 생성하지 못할 수 있다.
- core SVG와 native PNG는 좌표 반올림으로 1px 내외 크기 차이가 날 수 있으므로 diff 전 정규화 기준이 필요할 수 있다.
- render tree JSON이 큰 문서에서는 매우 커질 수 있으므로 출력 위치와 파일 크기 안내가 필요하다.
