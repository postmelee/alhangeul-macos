# Issue #33 수행 계획서

## 타스크

- GitHub Issue: #33
- 마일스톤: M050
- 제목: Quick Look thumbnail smoke test 실패 원인 분석

## 배경

Issue #27 smoke test에서 다음 상태를 확인했다.

- `/Users/melee/Applications/알한글.app` 설치와 PlugInKit 등록은 성공했다.
- Quick Look preview/thumbnail extension 표시명은 `알한글 미리보기`, `알한글 썸네일`로 노출됐다.
- 샘플 `KTX.hwp`의 UTI는 `com.haansoft.hancomofficeviewer.mac.hwp`로 확인됐고, extension의 `QLSupportedContentTypes`에 포함돼 있다.
- `pluginkit -e use` 이후에도 `qlmanage -t`는 `No thumbnail created`로 종료됐다.
- `qlmanage -m plugins`에서는 `rhwp`, `hwp`, `알한글`, `postmelee` 관련 항목이 노출되지 않았다.

2026-04-25 기준으로 `devel`은 PR #38까지 반영된 `22aa57f`이며, #26/#27/#37 결과가 포함된 상태에서 조사를 이어간다. 테스트용 파일은 작업지시자 지정에 따라 `/Users/melee/Documents/projects/rhwp-mac/samples` 아래 파일을 사용한다.

## 목표

Quick Look discovery, PlugInKit 등록, LaunchServices 등록, Thumbnail extension 실행 경로를 분리해 실제 Finder thumbnail 생성 실패 원인을 좁힌다.

## 범위

- 현재 Xcode target 설정과 생성된 app/appex Info.plist 확인
- PlugInKit 등록 상태와 중복 등록 상태 확인
- `qlmanage -m plugins` 미노출 원인 조사
- `HwpThumbnailProvider`가 실제 실행되는지 확인
- extension principal class/module 로딩 가능성 확인
- signing, sandbox, LaunchServices, Quick Look cache 계층 중 어느 지점에서 실패하는지 분리
- 필요한 경우 최소 설정 또는 코드 수정안 작성

## 제외

- `알한글` 표시명/배포명 정리 자체는 Issue #27 범위로 유지한다.
- release notarization, DMG, GitHub Release 자동화는 별도 release pipeline 이슈 범위로 유지한다.
- `Vendor/rhwp` core 수정은 이번 원인 분석에서 필요한 증거가 나오기 전에는 진행하지 않는다.

## 접근 방식

1. 정적 설정을 먼저 확인한다.
   - `project.yml`
   - HostApp/QLExtension/ThumbnailExtension Info.plist
   - 생성된 `RhwpMac.xcodeproj` build setting
   - package 산출물의 app/appex Info.plist
2. 등록 계층을 확인한다.
   - LaunchServices 등록 앱 경로
   - PlugInKit extension 경로와 enable 상태
   - 중복 등록 또는 예전 `RhwpMac.app` 잔존 여부
3. 실행 계층을 확인한다.
   - `qlmanage -m plugins`
   - `qlmanage -t`
   - unified log에서 extension discovery, 로딩, crash, sandbox 거부 여부
4. 실패 지점을 기준으로 최소 수정안을 판단한다.
   - 설정 문제면 `project.yml`/Info.plist 수정
   - extension 실행 문제면 provider instrumentation 또는 코드 수정
   - 운영 절차 문제면 manual 문서만 보정

## 리스크

- Quick Look/LaunchServices 상태는 사용자 환경 캐시와 설치본 중복에 영향을 받는다.
- sandbox 또는 서명 정책 문제는 `qlmanage` 명령만으로 재현 정보가 부족할 수 있다.
- #26/#27/#37은 `devel`에 merge된 상태이므로, 실제 조사는 `devel` 최신 기준을 반영한 `/tmp/rhwp-mac-task33`의 `local/task33` worktree에서 진행한다.
- 사용자 환경에는 예전 `RhwpMac.app` 또는 중간 산출물이 LaunchServices/PlugInKit 캐시에 남아 있을 수 있으므로, 현재 설치본과 stale 등록 항목을 분리해 기록한다.
