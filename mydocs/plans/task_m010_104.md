# Issue #104 수행 계획서

## 작업명

rhwp v0.7.9 stable tag 반영 및 앱 검증 버전 등록

## 이슈

- GitHub Issue: #104
- Milestone: v0.1
- 문서 prefix: `task_m010_104`
- 이슈 URL: https://github.com/postmelee/alhangeul-macos/issues/104

## 목적

앱 저장소가 참조하는 `edwardkim/rhwp` core를 정식 release tag `v0.7.9` 기준으로 갱신한다.

완료 후 `RustBridge/Cargo.toml`, `RustBridge/Cargo.lock`, `rhwp-core.lock`, Rust bridge 산출물은 모두 `v0.7.9` release tag와 resolved commit `0fb3e6758b8ad11d2f3c3849c83b914684e83863` 기준으로 일치해야 한다. 또한 Quick Look preview, Finder thumbnail, HostApp viewer를 테스트할 수 있도록 Release package 산출물을 표준 Applications 경로에 등록하고 smoke 결과를 남긴다.

## 배경

현재 저장소는 `rhwp v0.7.8` Stable release tag pin 상태다. 2026-04-30에 upstream `edwardkim/rhwp`의 `v0.7.9` release가 배포되었고, 사전 확인에서 `./scripts/update-rhwp-core.sh --check --channel stable --tag v0.7.9`가 통과했다.

`v0.7.9`는 `v0.7.8` 후속 패치 사이클이며, release note 기준으로 cell padding 방어 로직 보정과 외부 PR cherry-pick을 포함한다. 앱 저장소는 core source 자체를 수정하지 않고, release tag pin과 Rust bridge 산출물 provenance를 갱신해 macOS viewer/extension 경로에서 기존 C ABI와 Swift renderer가 유지되는지 확인한다.

이번 요청에는 "quicklook, thumbnail 및 viewer를 테스트할 수 있도록 릴리즈 버전을 응용프로그램에 등록"하는 작업도 포함된다. 따라서 Debug build만으로 끝내지 않고, `build_run_guide.md` 기준 Release package 산출물을 `$HOME/Applications/AlhangeulMac.app`에 설치 및 LaunchServices/PlugInKit 등록한 뒤 Quick Look/Thumbnail/Viewer smoke를 수행한다.

## 설계 방향

Stable 기준은 `release tag + resolved commit`으로 고정한다. `RustBridge/Cargo.toml`은 branch나 floating ref를 사용하지 않고 `tag = "v0.7.9"`로 전환한다. `RustBridge/Cargo.lock`에는 Cargo가 해석한 실제 source commit을 남기고, `rhwp-core.lock`에는 `rhwp_ref_kind = "release-tag"`, `rhwp_release_tag = "v0.7.9"`, resolved commit, artifact hash/size를 함께 기록한다.

앱 쪽 ABI와 renderer contract는 기존 PageRenderTree 기반 경로를 유지한다. Rust bridge는 기존 `rhwp_*` C ABI를 유지하고, Swift `RhwpCoreBridge`, `Shared`, `HostApp`, `QLExtension`, `ThumbnailExtension`은 dependency 갱신으로 인해 필요한 최소 적응이 있을 때만 변경한다.

검증용 앱 등록은 `build_run_guide.md`의 Finder 통합 확인 절차를 따른다. Debug `CODE_SIGNING_ALLOWED=NO` 산출물은 compile/link 확인용으로만 사용하고, Quick Look/Thumbnail 등록 판정은 Release package 산출물을 `$HOME/Applications/AlhangeulMac.app`에 설치한 뒤 `lsregister`, `pluginkit`, `qlmanage`로 확인한다.

버전 등록은 앱 source plist와 release package 입력 version이 서로 어긋나지 않게 처리한다. 기존 `0.1.0`과 구분이 필요한 경우에는 `Sources/HostApp/Info.plist`, `Sources/QLExtension/Info.plist`, `Sources/ThumbnailExtension/Info.plist`의 bundle version 값을 같은 기준으로 조정하고, release script의 version 검증 기준을 깨지 않도록 한다.

## 리스크

- `v0.7.9` tag 자체는 required API gate를 통과했지만, transitive dependency 해석이나 generated artifact hash/size는 실제 build 후에만 확정된다.
- Rust bridge build는 network fetch 또는 local Cargo cache 상태에 영향을 받을 수 있다. 실패 시 release lookup failure, dependency fetch failure, compile failure를 분리해 보고한다.
- core render tree JSON shape가 `v0.7.8`과 달라졌다면 Swift decoder나 renderer smoke에서 실패할 수 있다. 이 경우 core 전환 문제인지 Swift 적응 필요 사항인지 분리해야 한다.
- 앱/extension bundle version을 조정할 경우 HostApp, Quick Look, Thumbnail plist가 서로 다른 버전으로 남지 않게 해야 한다.
- Release package 산출물을 `$HOME/Applications/AlhangeulMac.app`에 설치하는 검증은 기존 표준 설치본을 교체한다. 이 경로 밖의 오래된 앱 후보는 임의 삭제하지 않는다.
- Quick Look preview GUI 확인은 환경에 따라 자동화가 어렵다. `qlmanage -t`와 PlugInKit 등록 확인으로 대체한 항목은 보고서에 명확히 남긴다.
