# Task M015 #108 수행계획서

## 목적

Swift native renderer가 rhwp core v0.7.9 PageRenderTree의 도형 children 구조를 올바르게 그리도록 보강한다.

완료 후 `samples/basic/BookReview.hwp` 첫 페이지에서 `Rectangle` children 아래의 텍스트가 HostApp, Quick Look, Thumbnail이 공유하는 native render 경로에서 보이는 상태가 되어야 한다.

## 배경

현재 제품 렌더 경로는 `rhwp_render_page_tree`가 반환하는 PageRenderTree JSON을 Swift `RenderTree.swift`가 디코딩하고 `CGTreeRenderer.swift`가 CoreGraphics로 그린다.

rhwp core v0.7.9 기준 PageRenderTree에는 `Rectangle`, `Line`, `Ellipse`, `Path`, `Image` 같은 leaf 성격 노드에도 children이 붙을 수 있다. `samples/basic/BookReview.hwp`는 텍스트가 `Rectangle` children 아래에 존재하지만, 현재 Swift renderer는 도형 노드를 그린 뒤 children을 순회하지 않아 텍스트가 보이지 않는다.

core의 PageLayerTree builder는 own paint op를 leaf로 먼저 배치하고 children을 뒤에 이어 붙이는 순서를 사용한다. 이번 작업은 PageLayerTree 전환이 아니라 기존 PageRenderTree 기반 Swift renderer에서 이 순회 정책을 맞추는 작업이다.

## 범위

포함:

- `CGTreeRenderer`의 도형/이미지 노드 렌더 후 children 순회 정책 정리
- `Rectangle`, `Line`, `Ellipse`, `Path`, `Image` children 렌더링 보강
- own node draw 후 children draw 순서가 core PageLayerTree builder와 맞는지 검토
- 도형 내부 텍스트가 필요한 샘플에서 보이는지 확인
- `BookReview.hwp` 중심의 render smoke와 비교 산출물 기록

제외:

- PageLayerTree ABI 추가 또는 기본 렌더 경로 전환
- 이미지 `crop/effect/brightness/contrast` 보강
- Equation/RawSvg 범용 렌더링
- 한컴과 완전 동일한 텍스트 효과 parity
- core `rhwp` upstream 수정

## 설계 방향

기존 Swift renderer의 구조를 유지하고, 도형 계열 노드가 자기 자신을 그린 뒤 children을 계속 순회하도록 작은 변경으로 해결한다.

우선순위는 `Rectangle` children 텍스트 누락을 해결하는 것이다. `Line`, `Ellipse`, `Path`, `Image`도 PageRenderTree contract상 children을 가질 수 있으므로 같은 정책을 적용하되, clipping이 필요한 특수 케이스는 이번 작업에서 과도하게 일반화하지 않는다.

렌더 순서는 core PageLayerTree의 `LayerBuilder`와 같이 own leaf를 먼저 그리고 children을 뒤에 그리는 방향으로 둔다. 기존 `Body.clipRect`, `TableCell.clip`, `Group` 처리와 충돌하지 않도록 현재 clipping 지점을 유지한다.

## 리스크

- 도형 children을 일괄 순회하면 일부 문서에서 기존보다 draw order가 바뀌어 겹침이 드러날 수 있다.
- 도형 내부 clipping 정책이 core SVG와 완전히 같지 않으면 텍스트가 보이더라도 경계 밖 표시 차이가 남을 수 있다.
- 이미지 children을 순회할 경우 이미지 위/아래 텍스트 순서가 샘플별로 다르게 보일 수 있어 core PageLayerTree 순서와 비교가 필요하다.
- Xcode build와 render smoke는 로컬 Xcode/Simulator/Quick Look 상태에 영향을 받을 수 있다.
