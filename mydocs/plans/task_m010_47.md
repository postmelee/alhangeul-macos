## 목적

Task #45 (PR #46)에서 `CLAUDE.md`를 `@AGENTS.md` 임포트 표기 + 안내 1줄로 단순화한 변경이 Claude Code 새 세션에서 실제로 의도대로 동작하는지 실측하고, 결과에 따라 현재 구조를 유지하거나 폴백 `CLAUDE.md`를 작성한다.

## 배경

- Task #45 최종 보고서(`mydocs/report/task_m010_45_report.md`) "잔여 위험 1번"에서 `@AGENTS.md` 임포트의 실제 적용 여부는 새 세션을 직접 열어야 측정 가능하다고 기록되었다.
- Codex 측은 `AGENTS.md`를 1차 적재 진실 원천으로 삼고 있어 본 임포트 동작과 무관하게 기존 운영이 가능하다.
- Claude Code 측은 `CLAUDE.md`를 1차로 적재한다. Task #45 변경 이후의 `CLAUDE.md`는 본문 4줄 + `@AGENTS.md` 한 줄로만 구성되어 있어, 임포트 미적용 시 운영 규칙이 사실상 비어 있는 상태가 된다.
- 본 타스크는 운영 문서/세션 적재만 다루며, 코드/빌드 산출물은 변경하지 않는다.

## 범위

### 포함

- 본 저장소 루트(`/Users/melee/Documents/projects/rhwp-mac`)에서 시작된 Claude Code 세션 시스템 프롬프트에 `AGENTS.md` 본문이 포함되는지 실측한다.

  - 미적용: `CLAUDE.md`를 짧은 안내문 + `AGENTS.md` 핵심 규칙 요약 폴백으로 재작성한다. 폴백은 Task #45가 정의한 핵심 강제 규칙·필수 참조 문서 인덱스를 줄이지 않는다.

## 설계 방향

- 본 세션이 곧 측정 대상이다. 시스템 프롬프트 적재 본문에 `Contents of /Users/melee/Documents/projects/rhwp-mac/AGENTS.md`로 시작하는 블록이 존재하는지 1차 확인한다.
- 1차 확인 결과를 단일 진실로 삼되, 보고서에는 측정 시각·세션 식별 가능한 단서·관측 본문 길이를 함께 적는다.
- 폴백 작성이 필요한 경우 다음 원칙을 적용한다.
  - `CLAUDE.md`는 단일 화면(80~120줄) 내에서 핵심 강제 규칙과 필수 참조 문서 인덱스를 모두 노출.
  - `AGENTS.md`는 `mydocs/manual/` 매뉴얼 인덱스의 단일 진실 원천을 유지하고, `CLAUDE.md`는 그 요약본임을 머리말에 명시.
  - 동일 규칙을 두 문서가 표현하므로 향후 변경 시 동기화 책임이 발생함을 안내문에 기재.
