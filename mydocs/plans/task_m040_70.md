- 브랜치: `local/task70`
- 기준 브랜치: `devel`

## 목적

HostApp viewer의 왼쪽 디버깅 사이드바를 제거하고, 앱/빌드/확장 정보를 macOS 표준 흐름인 `알한글 > 알한글에 관하여`에서 확인할 수 있도록 정리한다. 확장 등록 상태는 번들 ID 문제와 조회 실패를 구분해 사용자에게 오해가 적은 상태로 표시한다.

## 배경

현재 `ContentView`의 사이드바는 문서 정보, Quick Look/Thumbnail 확장 등록 상태, 빌드 버전을 함께 표시한다. 문서명, 현재 쪽수, 확대율은 하단 상태바에도 이미 노출되므로 사이드바의 확장/빌드 정보는 제품 화면보다 디버깅 정보 성격이 강하다.

추가 확인 결과 Quick Look 미리보기와 Thumbnail 확장의 번들 ID는 `com.postmelee.alhangeulmac.QLExtension`, `com.postmelee.alhangeulmac.ThumbnailExtension`으로 빌드 산출물과 시스템 등록 레코드가 일치한다. 다만 샌드박스된 앱 내부에서 `/usr/bin/pluginkit -m`을 `Process`로 실행하는 현재 조회 방식은 실패할 수 있고, 이때 앱 화면에는 `확인할 수 없음`으로 표시된다.

## 범위(포함·제외)

포함 범위:

- HostApp viewer 루트 UI에서 왼쪽 디버깅 사이드바 제거
- `알한글 > 알한글에 관하여` 메뉴 항목과 About 정보 화면 추가
- 앱 버전, 빌드 번호, Quick Look 미리보기 확장, Thumbnail 확장 정보 표시
- 확장 번들 포함 정보와 시스템 등록 조회 결과를 구분하는 표시 정책 정리
- 샌드박스 환경에서 등록 조회 실패 시 원인이 불명확한 `확인할 수 없음`보다 명확한 상태 문구 제공

제외 범위:

- Quick Look/Thumbnail 렌더링 동작 자체 변경
- Finder/LaunchServices/PlugInKit 캐시 초기화 절차 자동화
- release, signing, notarization, DMG, Homebrew Cask 변경
- `Sources/RhwpCoreBridge`와 Rust FFI 경계 변경
- rhwp core dependency 변경

## 설계 방향

- `ContentView`는 문서 viewer 중심으로 단순화하고, 문서명/쪽수/확대율은 기존 하단 상태바와 toolbar에 유지한다.
- About 화면은 HostApp 소유 영역에 별도 SwiftUI view와 작은 AppKit window bridge를 두는 방향을 우선 검토한다. macOS 12 타깃을 유지하므로 `openWindow` 의존보다 `NSWindowController` + `NSHostingController`가 호환성 면에서 안정적이다.
- `BuildInfo`와 `ExtensionStatusModel`은 기존 분리 구조를 유지하되 About 화면에서 재사용할 수 있게 정리한다.
- 확장 상태는 최소한 `내장됨`, `시스템 등록됨`, `시스템 등록 확인 불가`를 구분한다. 샌드박스에서 `pluginkit` 조회가 실패하는 경우를 등록 실패로 오인하지 않도록 표시한다.
- HostApp에서 AppKit 사용은 허용하되 `Sources/RhwpCoreBridge`에는 AppKit/UIKit 의존을 추가하지 않는다.

## 잠정 단계(3~6단계)

1. 현재 UI와 상태 조회 경로 정리
   - 사이드바 제거 후 남겨야 할 문서 정보 위치를 확정한다.
   - About 화면에 필요한 앱/확장 데이터 모델을 확정한다.
2. About 메뉴와 정보 화면 구현
   - 앱 메뉴의 About 항목을 교체하거나 보강한다.
   - 앱 버전, 빌드 번호, 확장 이름, 번들 ID, 내장 여부를 표시한다.
3. 확장 등록 상태 표시 보정
   - `pluginkit` 조회 실패와 실제 미등록 상태를 구분한다.
   - 샌드박스 환경에서 신뢰 가능한 fallback 문구를 제공한다.
4. 검증과 보고
   - 빌드, 정적 검사, 메뉴 동작 확인을 수행한다.
   - 단계별 완료보고서와 최종 결과보고서에 결과를 기록한다.

## 리스크

- `pluginkit` 조회는 macOS 버전, 앱 샌드박스, 설치 위치, PlugInKit 캐시에 영향을 받으므로 앱 내부에서 항상 동일한 결과를 보장하기 어렵다.
- About 메뉴를 커스텀 창으로 구현하면 표준 About 패널과 다른 창 수명 관리가 필요하다.
- 사이드바 제거 후 빈 문서 상태와 문서 열기 흐름이 어색해질 수 있어 첫 화면과 toolbar 동작을 함께 확인해야 한다.
- Xcode project 원본은 `project.yml`이므로 필요한 경우 `xcodegen generate` 경로를 준수해야 한다.
