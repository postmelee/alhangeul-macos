# Issue #55 수행 계획서

## 작업명

release tag dependency 전환을 위한 core API compatibility와 update architecture 정리

## 이슈

- GitHub Issue: #55
- Milestone: v0.1.0
- 문서 prefix: `task_m010_55`
- 이슈 URL: https://github.com/postmelee/alhangeul-macos/issues/55

## 목적

`RustBridge`를 `edwardkim/rhwp` release tag 기반 dependency로 전환하기 전에, 앱이 요구하는 core API contract와 release update compatibility gate를 명확히 정리한다.

완료 후 Issue #30 수행자는 branch나 floating ref가 아니라 release tag와 resolved commit을 기준으로 target core를 선택하고, 실패 원인을 API 누락, lock 불일치, artifact 불일치, render smoke 실패로 분리해 판단할 수 있어야 한다.

## 배경

Issue #30은 `Vendor/rhwp` submodule을 제거하고 `RustBridge`를 release tag 기반 dependency로 전환하는 작업이다. 하지만 2026-04-26 확인 기준 최신 release `v0.7.3`의 resolved commit `c2e8a3461de800a02f76127ff4797bade1d4e532`로 `RustBridge`를 빌드하면 현재 native renderer가 필요한 API가 없어 실패한다.

확인된 핵심 누락 API는 다음이다.

- `DocumentCore::build_page_render_tree`
- `DocumentCore::get_bin_data`

따라서 #30을 바로 진행하기 전에 현재 bridge가 기대하는 API 표면, release tag compatibility gate, update script 구조, lock 정합성 기준을 문서화해야 한다.

## 설계 방향

이번 작업은 release tag 전환 자체가 아니라 전환 가능 여부를 판단하는 기준을 세우는 작업이다. 따라서 앱 저장소의 현재 native render tree 경로를 기준 경로로 유지하고, release tag가 그 경로를 만족하는지 검증하는 문서와 체크리스트를 만든다.

`RustBridge`는 유일한 core adapter로 유지한다. Swift 계층은 `DocumentCore` 내부 struct나 Rust crate 세부 구현에 직접 반응하지 않고, C ABI contract만 사용하도록 경계를 설명한다.

release tag compatibility gate는 다음 실패 유형을 구분하도록 설계한다.

- `missing core API`
- `Cargo.lock mismatch`
- `artifact hash mismatch`
- `FFI symbol diff`
- `render smoke failure`

`main`, `devel` 같은 branch는 필요한 API가 포함된 과도기 commit을 확인하는 참고 출처로만 다루고, 앱의 안정 기준은 release tag + resolved commit으로 제한한다.

## 리스크

- 최신 release tag 정보는 시간이 지나면 바뀔 수 있으므로, 구현 단계에서 `edwardkim/rhwp`의 실제 latest release와 resolved commit을 다시 확인해야 한다.
- v0.7.3의 실패 원인을 과도하게 일반화하면 후속 release의 다른 실패 유형을 놓칠 수 있다. API 누락과 build/script/lock 실패를 분리해 기록한다.
- update script 설계가 실제 구현처럼 상세해지면 #30 범위를 침범할 수 있다. 이번 작업은 gate와 실패 처리 기준을 정리하는 수준으로 제한한다.
- fallback 경로를 문서화할 때 제품 기준 경로가 흐려질 수 있다. native render tree를 기준 경로로 두고 SVG fallback은 진단/임시 후보로만 취급한다.
