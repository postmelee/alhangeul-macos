# Issue #76 수행 계획서

## 작업명

rhwp PR #385 반영 core pin 갱신과 native bridge 검증

## 이슈

## 목적

`edwardkim/rhwp` PR #385로 공식 반영된 native `PageRenderTree` bridge API를 alhangeul-macos의 실제 use case 기준으로 검증하고, 현재 Demo/Preview core pin을 upstream merge commit 기준으로 갱신한다.

완료 후 앱 저장소는 `DocumentCore::build_page_render_tree`, `DocumentCore::get_bin_data`가 포함된 upstream 공식 commit을 `RustBridge` dependency와 lock provenance에 기록하고, HostApp/Quick Look/Thumbnail이 사용하는 native render tree 경로가 계속 동작함을 검증 결과로 설명할 수 있어야 한다.

## 배경

현재 alhangeul-macos는 `rhwp-core.lock`과 `RustBridge/Cargo.toml`에서 `edwardkim/rhwp` commit `1e9d78a1d40c71779d81c6ec6870cd301d912626`을 Demo/Preview용으로 고정하고 있다. 이 commit은 alhangeul-macos에서 먼저 검증한 native render tree bridge API를 포함하지만, upstream의 공식 PR merge commit은 아니다.

`edwardkim/rhwp` PR #385는 Task #363 처리로 cherry-pick merge 되었고, 메인테이너는 `PageRenderTree` native bridge API가 release contract로 노출되었으니 alhangeul-macos use case 검증 결과를 반영해 달라고 요청했다.

확인된 기준:

- upstream PR: https://github.com/edwardkim/rhwp/pull/385
- upstream issue: https://github.com/edwardkim/rhwp/issues/363
- upstream merge commit: `e91ecea3174a0da0ad7a1ea495cacc4f8772c31d`
- 현재 앱 저장소 pin: `1e9d78a1d40c71779d81c6ec6870cd301d912626`
- 최신 확인 release `v0.7.7`은 `DocumentCore::build_page_render_tree`가 없어 Stable release tag 전환 기준을 아직 충족하지 못한다.

따라서 이번 작업은 Stable release tag 전환이 아니라 Demo/Preview commit pin 갱신과 native bridge 검증으로 제한한다.

## 설계 방향

기준 경로는 기존 native render tree 경로를 유지한다. `RustBridge`는 `DocumentCore::build_page_render_tree` 결과를 JSON으로 직렬화해 `rhwp_render_page_tree`로 반환하고, Swift는 `RenderTree.swift` 모델로 decode한다. 이미지 bytes는 render tree JSON에 직접 포함하지 않고 `bin_data_id`를 `rhwp_image_data`로 조회하는 현재 contract를 유지한다.

core dependency 기준은 release tag 안정 기준과 Demo/Preview 기준을 분리한다. `v0.7.7`이 required API를 만족하지 못하는 동안 Stable 전환은 blocked로 남기고, 이번 작업에서는 upstream merge commit을 full SHA `rev`로 고정한다.

검증은 pin 갱신 자체보다 use case 확인에 초점을 둔다. 최소한 RustBridge build, lock verify, C ABI symbol diff, HostApp build, render smoke를 통과해야 하며, 이미지가 있는 샘플에서 `bin_data_id` 조회 실패가 없는지 확인한다.

## 리스크

- upstream `devel` merge commit은 release tag 안정 기준이 아니므로 Stable로 표시하면 안 된다.
- `e91ecea`는 현재 pin보다 많은 upstream 변경을 포함하므로 render 결과 차이가 발생할 수 있다. 단순 dependency 갱신이 아니라 smoke 검증 결과를 함께 남긴다.
- `v0.7.7` 이후 새 release가 나올 수 있으므로, 구현 단계 시작 시 latest release를 다시 확인해야 한다.
- render tree JSON shape가 Swift decoder 기대와 달라진 경우 HostApp/Quick Look/Thumbnail 렌더가 실패할 수 있다.
- 이미지 bytes 조회의 1-indexed `bin_data_id`와 0-indexed core `get_bin_data` 변환 규칙이 깨지면 이미지 렌더 누락이 발생할 수 있다.
- Rust bridge 산출물은 크고 재생성 시간이 길 수 있다. 실패 시 dependency fetch, build failure, artifact mismatch를 분리해 보고한다.
