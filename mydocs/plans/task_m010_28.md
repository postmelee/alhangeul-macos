## 배경

현재 render smoke test와 일부 문서는 `Vendor/rhwp/samples` 아래 샘플 파일을 기본 검증 입력으로 사용한다. 후속 작업에서 `Vendor/rhwp` submodule을 제거하거나 `RustBridge`를 git rev dependency로 전환하면, 기본 검증 명령과 Quick Look/Thumbnail smoke test 문서가 샘플 경로 때문에 깨질 수 있다.

확인된 직접 의존 지점:

- `scripts/validate-stage3-render.sh`
  - `Vendor/rhwp/samples/basic/KTX.hwp`
  - `Vendor/rhwp/samples/basic/request.hwp`
  - `Vendor/rhwp/samples/exam_kor.hwp`
- `README.md`
- `mydocs/manual/build_run_guide.md`
- `mydocs/manual/release_distribution_guide.md`

## 현재 추가 확인

작업지시자가 이전에 `Vendor/rhwp/samples` 파일을 저장소 루트의 `samples/`로 직접 추가해 둔 상태를 확인했다.

- 현재 `samples/`는 Git 기준 untracked 상태다.
- 파일 수: 180개
- 크기: 약 60MB
- #28의 대표 3개 샘플은 이미 존재한다.
  - `samples/basic/KTX.hwp`
  - `samples/basic/request.hwp`
  - `samples/exam_kor.hwp`
- 세 파일은 `Vendor/rhwp/samples`의 원본과 sha256이 동일하다.

따라서 새 `Samples/RenderSmoke/` 경로를 만들지 않고, 기존 루트 `samples/`를 앱 저장소가 소유하는 fixture 경로로 채택한다.

## 목표

- 앱 저장소 루트의 `samples/`를 render smoke fixture 경로로 공식화한다.
- 기본 render smoke test가 `Vendor/rhwp` 없이도 설명 가능하고 실행 가능한 경로를 사용하게 한다.
- Quick Look/Thumbnail smoke test 문서에서 기본 fixture 예시와 사용자 파일 예시를 분리한다.
- 샘플 파일 provenance와 재배포 판단 근거를 문서로 남긴다.

## 주요 판단

### fixture 경로

기본 fixture 경로는 루트 `samples/`로 둔다.

이유:

- 작업지시자가 이미 `Vendor/rhwp/samples`를 루트 `samples/`로 추가했다.
- 기존 문서와 과거 smoke test 기록에도 `/samples` 경로가 일부 등장한다.
- `Vendor/rhwp` 제거 후에도 동일한 상대 경로로 render/Quick Look/Thumbnail smoke test를 설명할 수 있다.
- 별도 `Samples/RenderSmoke/`를 만들면 이미 추가된 샘플과 중복 경로가 생긴다.

### 샘플 재배포 판단

우선 `Vendor/rhwp`의 MIT license와 샘플 파일 commit provenance를 확인한다. 루트 `samples/`는 `Vendor/rhwp/samples`와 동일한 복사본으로 확인되었으므로, provenance 문서에는 원본 submodule commit, 파일 hash, 사용 목적을 기록한다.

개별 HWP 파일의 원출처가 저장소 license만으로 충분히 설명되지 않는 경우에는 provenance 문서에 제한 사항을 명시하고, 이번 PR에서는 “기존 core samples 복사본을 앱 저장소 검증 fixture로 고정한다”는 범위로 다룬다.
