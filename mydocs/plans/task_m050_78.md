# Issue #78 수행 계획서

## 목적

Apple Developer Program 가입 이후 실제 준비된 Developer ID Application 인증서와 `notarytool` keychain profile 기준으로 서명, 공증, DMG 배포 절차 문서를 최신화한다.

## 배경

Issue #78은 Apple Developer Program 가입과 Developer ID Application 인증서 설치, `notarytool` profile 검증이 완료된 뒤 등록된 문서 갱신 작업이다.

확인된 비밀이 아닌 운영 값은 다음과 같다.

- Team ID: `XH6JHKYXV8`
- Developer ID Application identity: `Developer ID Application: Taegyu Lee (XH6JHKYXV8)`
- notarytool keychain profile: `alhangeul-notary`

현재 `mydocs/manual/release_distribution_guide.md`에는 Apple Developer credential이 아직 준비되지 않았다는 전제와 public release 실행 전 확인 항목이 섞여 있다. 이 문서를 실제 운영 상태와 맞추되, app-specific password, Apple ID password, `.p8`, `.p12` 같은 민감 값은 문서와 commit에 남기지 않는다.

## 범위

포함:

- `release_distribution_guide.md`의 Apple Developer Program 준비 상태 갱신
- Developer ID Application signing identity와 `notarytool` keychain profile 사용 방식 정리
- public DMG 생성 전 확인, 공증 제출, staple, Gatekeeper 검증 흐름 보강
- 비밀값 관리 원칙과 작업지시자가 제공해야 하는 비밀이 아닌 운영 값 정리
- 필요 시 README의 release packaging 안내와 매뉴얼 링크 표현 보정

제외:

- 실제 public release 실행
- GitHub Release 생성과 asset upload
- Homebrew Cask PR 생성 또는 외부 tap 배포
- 인증서, app-specific password, API private key, `.p12` 등 비밀값 기록
- Xcode project 직접 수정

## 설계 방향

릴리스/배포 절차의 진실 원천은 `mydocs/manual/release_distribution_guide.md`로 유지한다. 공개 README에는 상세 credential 절차를 늘리지 않고, 필요한 경우 release packaging 안내가 매뉴얼로 연결되도록만 보정한다.

문서에는 재현 가능한 public release 입력값 중 비밀이 아닌 값만 적는다. 실제 secret은 macOS Keychain, GitHub Secrets, 작업지시자 로컬 환경에 보관하는 원칙을 명확히 한다.

기존 `scripts/release.sh`가 요구하는 환경변수와 문서의 명령 예시를 일치시킨다. 스크립트 변경은 문서와 실제 입력값이 어긋나는 문제가 확인될 때만 최소 범위로 검토한다.

## 검증 계획

- `rg --line-number 'Apple Developer|Developer ID|notarytool|notarization|ALHANGEUL|공증|서명' README.md mydocs/manual/release_distribution_guide.md scripts/release.sh`
- `bash -n scripts/release.sh scripts/package-release.sh`
- `./scripts/release.sh --help`
- `git diff --check`

실제 public release, 공증 제출, Gatekeeper 검증은 이번 문서 갱신 범위에서 실행하지 않는다. 필요한 경우 별도 승인 후 후속 단계 또는 후속 이슈로 분리한다.

## 리스크

- 인증서와 keychain profile은 로컬 사용자 환경에 있으므로 문서 검증만으로 실제 공증 성공을 보장할 수 없다.
- App Store Connect Team API Key 기반 인증을 추가로 선택하면 Apple ID app-specific password 방식과 문서 분기가 필요할 수 있다.
- `scripts/release.sh` public mode는 clean worktree를 요구하므로, 실제 release 실행은 별도 release 작업에서 수행해야 한다.
- Homebrew Cask의 실제 `sha256`은 signed/notarized public DMG 생성 후에만 확정된다.
