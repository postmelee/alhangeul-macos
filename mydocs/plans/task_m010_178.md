## 목적

GitHub Pages 홍보 페이지의 두 번째 섹션을 보강해, 첫 화면 이후 제품 가치가 더 자연스럽게 이어지도록 한다.

현재 두 번째 섹션은 `알한글` 제목 위 여백이 상대적으로 크게 느껴지고, 철학 설명이 일반 회색 본문처럼 보인다. 리드 문구 `HWP/HWPX 문서를 Mac에서 빠르게 확인하고, 편집하고, 공유하세요.`는 유지하되, 아래 철학 메시지를 더 강한 선언형 구조로 보여주고, 현재 스크린샷 이미지를 더 크게 배치해 제품 중심성을 높인다.

## 배경

기존 랜딩페이지 작업 #135에서 GitHub Pages용 홍보 페이지가 완성되었다. 이후 공개 페이지 확인 과정에서 두 번째 섹션의 시각 균형에 대한 추가 피드백이 나왔다.

- 첫 번째 섹션 이후 두 번째 섹션의 상단 진입 여백이 과해 보인다.
- 리드 문구의 타이포그래피는 유지할 만하지만, 철학 설명은 메시지 중요도에 비해 약하게 보인다.
- `og-main.png` 스크린샷은 제품 신뢰를 만드는 핵심 자산이므로 더 크게 보여줄 필요가 있다.

이번 작업은 기존 페이지의 전체 구조를 다시 설계하지 않고, 두 번째 섹션 안에서 spacing, copy presentation, screenshot scale을 좁게 보정한다.

## 범위

포함:

- `docs/index.html`의 두 번째 섹션 철학 설명 구조와 문구 보강
- `docs/styles.css`의 두 번째 섹션 상단 여백, 내부 간격, 스크린샷 폭 보정
- 기존 reveal timing에 영향이 있으면 `docs/script.js` 동작 확인
- 데스크톱과 모바일 뷰포트에서 두 번째 섹션 시각 검증
- 리드 문구 유지

제외:

- HostApp, Quick Look, Thumbnail extension, Swift/Rust 코드 변경
- GitHub Pages 저장소 설정 변경
- release artifact, 배포, 서명, 공증 변경
- 랜딩페이지 전체 재설계
- 새 이미지 자산 생성 또는 기존 이미지 교체
- 첫 번째 hero와 이후 Feature sticky 섹션의 구조 변경

## 설계 방향

두 번째 섹션은 기존 Apple-style white background, near-black text, muted gray, blue accent 톤을 유지한다. 새 장식이나 카드 그리드를 만들지 않고, 기존 페이지의 절제된 제품 소개 흐름 안에서 보강한다.

상단 여백은 `min-height`와 `align-items: center`의 중앙 정렬로 인해 커 보일 수 있으므로, 섹션을 상단 정렬에 가깝게 조정한다. 다만 첫 번째 섹션과 너무 붙어 보이지 않도록 데스크톱과 모바일의 padding을 별도로 확인한다.

철학 설명은 일반 본문보다 한 단계 높은 메시지로 보이게 한다. 후보 구조는 얇은 top/bottom hairline 안에 첫 문장을 선언형으로 강조하고, 두 번째 문장은 muted 설명으로 받는 방식이다.

스크린샷은 현재 `og-main.png`를 유지하면서 데스크톱 기준 최대 폭을 키운다. 이미지가 커진 만큼 하단 기능 4개 요약의 폭과 간격도 함께 확인해 섹션 전체가 흔들리지 않게 한다.

## 리스크

- 스크린샷을 키우면 두 번째 섹션의 세로 길이가 늘어나 기능 요약 영역이 아래로 밀릴 수 있다.
- 상단 여백을 줄이면 첫 섹션과 두 번째 섹션 사이 호흡이 부족해 보일 수 있다.
- 철학 설명을 강조하면 리드 문구보다 시각 위계가 과해질 수 있으므로, body보다 강하지만 lead보다 낮은 hierarchy를 유지해야 한다.
- 모바일에서는 긴 한국어 문장이 hairline 영역 안에서 과하게 줄바꿈될 수 있다.
- 기존 reveal animation이 섹션 내부 간격 변경 후 어색해질 수 있다.
