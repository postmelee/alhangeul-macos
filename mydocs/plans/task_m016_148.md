# Task #148 수행계획서: v0.1 서명·공증 배포 수준 결정

## 목적

M016의 선행 결정 항목인 v0.1 서명·공증 배포 수준을 확정한다. 결론은 현재 준비 상태 기준 `Developer ID signed + notarized DMG`를 public 기본값으로 두되, 개발용 zip과 rehearsal DMG, Homebrew Cask, App Store 후속 경로의 책임 경계를 명확히 분리하는 것이다.

## 배경

Issue #148은 unsigned, ad-hoc signed, Developer ID signed, notarized 배포 옵션을 비교하고 v0.1 최소 배포 수준을 결정하기 위해 등록됐다. milestone 16의 작업 순서상 #148이 #147, #145, #150보다 먼저 수행됐어야 했지만, 로컬 `local/task148` 브랜치에 Stage 3까지 진행된 뒤 PR 없이 중단되어 있었다.

2026-05-08 현재 `devel-webview`에는 #147, #145, #150, #167 결과가 이미 merge되어 있다. 따라서 이번 보정의 성격은 새 결정을 처음 작성하는 것이 아니라, 기존 #148 로컬 산출물을 현재 M016 문서명과 최신 release artifact 기준에 맞게 정리하고 후속 작업이 참조할 공식 판단으로 고정하는 것이다.

현재 저장소에는 다음 release 기반이 있다.

- `scripts/release.sh`: Developer ID 서명, app/DMG notarization, staple, Gatekeeper 검증, sha256 산출 경로를 포함한 public DMG pipeline
- `scripts/package-release.sh`: 개발·설치본 smoke용 Release zip package 생성
- `Casks/alhangeul-macos.rb`: public DMG를 바라보는 Homebrew Cask 초안
- `scripts/update-cask-sha256.sh`: public DMG `.sha256`에서 Cask version/sha256을 갱신하는 보조 스크립트
- `mydocs/manual/release_distribution_guide.md`: release artifact, signing/notarization, checksum/provenance, Homebrew Cask 운영 기준

## 범위

포함:

- unsigned, ad-hoc signed, Developer ID signed, Developer ID signed + notarized 배포 옵션 비교
- 현재 project signing, sandbox entitlement, extension bundle 구조가 public DMG와 Finder extension 등록에 미치는 영향 정리
- v0.1 public 기본 배포 수준을 `Developer ID signed + notarized DMG`로 확정
- 개발용 zip, rehearsal DMG, public DMG의 public 사용 가능 여부 분리
- Gatekeeper, quarantine, Quick Look/Thumbnail extension registration 관점의 사용자 안내 기준 정리
- README와 release guide의 설치 안내, checksum, Cask 기준을 현재 `Alhangeul.app` 산출물명과 맞춤
- Homebrew Cask는 public DMG 업로드와 sha256 고정 이후 배포 가능한 후속 경로로 정리

제외:

- 실제 `./scripts/release.sh <version>` public mode 실행과 notarization submission
- Git tag 생성, GitHub Release 생성, release asset upload
- Homebrew tap 생성, push, `Homebrew/homebrew-cask` 제출
- App Store Connect 제출, App Store signing/export, review 대응
- signing identity export, `.p12`, app-specific password, `.p8` 등 secret 생성·저장
- DMG artwork, auto update, Sparkle 등 별도 업데이트 체계 도입

## 설계 방향

- public 사용자가 받는 v0.1 artifact는 `scripts/release.sh <version>` public mode가 만드는 `alhangeul-macos-<version>.dmg`와 `.sha256`으로 한정한다.
- `scripts/release.sh --skip-notarize` rehearsal DMG와 `scripts/package-release.sh` zip은 layout, checksum, 설치본 smoke 입력으로만 사용하고 public release asset이나 Cask URL로 쓰지 않는다.
- Homebrew Cask는 같은 public DMG와 고정 sha256을 참조하는 보조 배포 경로로 유지하되, tap 선택과 게시 작업은 release 실행 시점의 별도 승인 사항으로 둔다.
- Mac App Store는 v0.1 범위 밖 후속 lane으로만 언급하고, 이번 타스크에서는 실제 제출 준비나 자동화를 추가하지 않는다.
- secret은 저장소에 기록하지 않고 Team ID, signing identity 표시명, notarytool profile name처럼 비밀이 아닌 운영 식별자만 문서화한다.
- 이미 merge된 #145 release artifact 기준과 충돌하지 않도록 #148 결론을 release guide의 정책 결정 근거로 맞춘다.

## 잠정 단계

### Stage 1: 현재 배포 자산과 credential 준비 상태 점검

- `scripts/release.sh`, `scripts/package-release.sh`, `Casks/alhangeul-macos.rb`, release guide, bundle signing 설정을 대조한다.
- Developer ID signing identity와 notarytool profile 확인 명령을 실행 가능한 범위에서 검증한다.
- 결과: 기존 로컬 작업에서 완료했고, M016 문서명으로 이관한다.

### Stage 2: v0.1 배포 수준과 사용자 안내 기준 확정

- unsigned/ad-hoc/Developer ID signed/notarized 옵션 비교를 문서화한다.
- v0.1 public 기본값을 `Developer ID signed + notarized DMG`로 둔다.
- Gatekeeper, checksum, Quick Look/Thumbnail extension 등록 안내 기준을 정리한다.
- 결과: 기존 로컬 작업에서 완료했고, 현재 `Alhangeul.app` 기준으로 보정한다.

### Stage 3: Homebrew Cask와 release 준비 자동화 보강

- Cask가 public DMG 기준을 바라보는지 확인한다.
- public DMG `.sha256`에서 Cask `version`/`sha256`을 갱신하는 guard script를 추가한다.
- Homebrew tap 운영은 후속 release 승인 사항으로 남긴다.
- 결과: 기존 로컬 작업에서 완료했고, #148 원래 범위에 맞게 “배포 실행”이 아니라 “Cask 준비 guard”로 축소 해석한다.

### Stage 4: M016 기준 정합화와 후속 범위 분리

- `task_m010_148*` 문서를 `task_m016_148*` 체계로 보정한다.
- README와 release guide의 오래된 `AlhangeulMac.app` 참조를 현재 `Alhangeul.app` 기준으로 수정한다.
- 기존 Stage 1-3 보고서의 파일명, 경로, 잔여 위험을 최신 merge 상태와 맞춘다.
- 2026-05-08 오늘할일에 #148 진행 상태를 기록한다.

### Stage 5: 최종 검증과 보고서 정리

- shell syntax, release help, Cask update guard, 문서 정합성, stale wording, whitespace를 검증한다.
- 최종 결과보고서에 v0.1 배포 수준 결정, 제외한 public release 실행, 후속 #151/#146 영향 범위를 남긴다.
- PR 생성 전 미커밋 파일이 없는지 확인한다.

## 리스크

- #148 결정보다 #145/#150 일부 작업이 먼저 merge되어 release 문서 일부가 선반영된 상태다. 이번 작업은 이를 되돌리지 않고 정책 근거를 뒤늦게 명확히 남긴다.
- public DMG가 아직 생성되지 않았으므로 Cask `sha256`은 실제값으로 고정할 수 없다.
- Notarization 최종 성공은 실제 public release 실행 시 Apple notarization server 판정에 의존한다.
- Homebrew tap 대상은 아직 확정하지 않는다. #148에서는 Cask source와 갱신 guard만 준비한다.
- App Store 배포는 별도 이슈로 분리하지 않으면 범위가 과도하게 커진다.
