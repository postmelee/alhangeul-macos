# Task M018 #199 수행계획서

## 목적

공식 릴리즈 설치본 `/Applications/Alhangeul.app`만 남긴 상태에서도 Finder thumbnail이 생성되지 않는 문제를 재현, 진단, 수정한다.

완료 후에는 HWP/HWPX 샘플에서 `qlmanage -t`가 시간 내 PNG를 생성하고, Finder icon view에서도 공식 `AlhangeulThumbnail` 확장이 멈추지 않아야 한다.

## 배경

중복 설치본, 오래된 테스트 빌드, LaunchServices/Quick Look 캐시는 정리했다. Spotlight 기준 설치본은 `/Applications/Alhangeul.app` 하나뿐이고, PluginKit에서도 `com.postmelee.alhangeul.ThumbnailExtension`이 활성화되어 있다.

그럼에도 다음 smoke가 완료되지 않는다.

```bash
qlmanage -t -s 256 -o /private/tmp/alhangeul-qlsingle.fUoR0v \
  /Users/melee/Documents/projects/rhwp-mac/samples/exam_science.hwp
```

`sample` 결과 공식 `AlhangeulThumbnail` 확장 내부의 `com.postmelee.alhangeul.thumbnail-render` 작업들이 `HwpPageImageRenderer.renderPage`에서 `CGContextFillRect` 이후 CoreGraphics 색 변환 mutex wait 상태에 머문다. 현재 `HwpThumbnailRenderCache`는 concurrent worker queue를 사용하므로 Finder의 다중 thumbnail 요청과 단일 smoke 요청 모두에서 렌더 경로가 안정적으로 완료되는지 확인해야 한다.

## 범위

포함:

- 공식 `/Applications/Alhangeul.app` 설치본 기준 thumbnail hang 재현 기록
- `Sources/ThumbnailExtension`과 `Sources/Shared` thumbnail 렌더 경로 분석
- CoreGraphics bitmap rendering 동시 실행 또는 색 공간 초기화 hang을 피하는 최소 수정
- HWP/HWPX `qlmanage -t` smoke 검증
- Finder/Quick Look smoke 기준에 필요한 문서 보강

제외:

- public release 생성, signing, notarization, DMG 게시
- Quick Look preview 전체 구조 변경
- `rhwp` core parser/renderer dependency 변경
