# Task M015 #107 수행계획서

## 목적

첫 출시 전 Swift native renderer 보강 작업에서 공통으로 사용할 샘플 smoke/diff 검증 세트를 정리한다.

완료 후 작업자는 `samples/basic/BookReview.hwp`와 `samples/복학원서.hwp`를 필수 smoke 대상으로 삼고, 도형 children, 이미지 effect/crop, placeholder/form, 스타일 보강의 대표 샘플을 같은 절차로 확인할 수 있어야 한다.

## 배경

M015 마일스톤은 HostApp, Quick Look, Thumbnail이 공유하는 Swift native renderer의 렌더 갭을 PageRenderTree 기반으로 줄이는 작업이다.

현재 저장소에는 기본 native render smoke인 `scripts/validate-stage3-render.sh`와 core SVG/native PNG 비교 진단 도구인 `scripts/render-debug-compare.sh`가 있다. 그러나 M015 세부 작업들이 어떤 샘플을 반드시 확인해야 하는지, 어떤 summary 값을 보고서에 기록해야 하는지, core SVG/native PNG/diff 산출물을 어디까지 보관해야 하는지가 아직 한 곳에 정리되어 있지 않다.

Task #108에서 `BookReview.hwp`가 도형 children 렌더링 회복 기준으로 쓰였고, 기존 작업들에서 `복학원서.hwp`는 page bbox overflow와 실제 제품 렌더 경로 확인에 반복 사용됐다. 이번 작업은 이런 대표 샘플과 산출물 기록 기준을 명시해 이후 renderer 보강 작업의 검증 품질을 맞춘다.

## 범위

포함:

- `samples/basic/BookReview.hwp`, `samples/복학원서.hwp`를 M015 필수 smoke 대상으로 정리
- 도형 children, 이미지 effect/crop, placeholder/form, 스타일 보강별 대표 샘플 후보 조사와 목록화
- `scripts/validate-stage3-render.sh` 기본 smoke와 `scripts/render-debug-compare.sh` core/native diff의 역할, 실행 기준, 출력 기준 문서화
- `NativeNonWhitePixels`, `TextRuns`, `HangulRuns`, `MissingHangulGlyphs`, core SVG/native PNG/diff 상태를 단계 보고서에 기록하는 기준 정리
- 생성 산출물의 로컬 보관 위치, PR/보고서에 남길 최소 메타데이터, 장기 보관 제외 기준 정리

제외:

- 새로운 렌더 엔진 도입
- PageLayerTree 전환 검증 세트 설계
- CI에서 모든 샘플을 매번 full diff하는 정책 결정
- rhwp core upstream 수정
- 한컴 viewer와의 픽셀 단위 자동 비교 도입

## 설계 방향

기존 진단 도구를 기준으로 문서와 작은 스크립트 보정을 우선한다. 새 검증 프레임워크를 만들기보다 `validate-stage3-render.sh`는 빠른 native smoke, `render-debug-compare.sh`는 특정 샘플의 core/native 원인 분해 도구로 유지한다.

필수 smoke 대상은 M015 렌더 보강의 공통 회귀 관문으로 문서화한다. 대표 샘플 목록은 기능 범주별로 관리하되, 모든 샘플을 매번 full diff하는 방식은 이번 작업에서 정하지 않는다.

산출물 보관은 저장소에 PNG/SVG를 커밋하는 방식이 아니라, 작업 단계별 로컬 출력 경로와 summary 핵심값을 보고서에 남기는 방식으로 정리한다. 필요한 경우에만 PR 설명 또는 피드백 문서에서 산출물 경로와 핵심 수치를 참조한다.

## 리스크

- `render-debug-compare.sh`의 선택 산출물인 core raster PNG와 diff PNG는 `qlmanage` 상태에 따라 생성되지 않을 수 있다. 이 경우 필수 산출물과 summary 기록을 기준으로 성공 여부를 판단해야 한다.
- `복학원서.hwp`는 core layout 한계가 섞인 샘플이므로, native renderer 회귀와 core 산출 한계를 구분하는 설명이 필요하다.
- 기본 smoke 샘플을 늘리면 로컬 검증 시간이 증가할 수 있다.
- 대표 샘플 목록을 과도하게 넓히면 M015 각 세부 작업의 단계 검증이 무거워질 수 있다.
