# Issue #102 수행 계획서

## 목적

저장소 PR에 대해 GitHub Copilot 자동 코드 리뷰를 적용하고, 이 저장소의 macOS/Swift/Rust bridge/하이퍼-워터폴 규칙을 Copilot 리뷰 지시사항으로 제공해 자동 리뷰 품질을 표준화한다.

## 배경

GitHub Copilot Code Review는 저장소의 custom instructions를 읽어 PR 리뷰에 반영할 수 있다. 현재 저장소에는 `.github/copilot-instructions.md`가 없고, GitHub Ruleset에도 Copilot 자동 리뷰 활성화 ruleset이 별도로 구성되어 있지 않다.

이 저장소는 `project.yml` 원본 정책, `Sources/RhwpCoreBridge`의 AppKit/UIKit 금지, Rust FFI 포인터/수명 규칙, core dependency lock 정합성, 하이퍼-워터폴 문서/PR 규칙처럼 일반 리뷰어가 놓치기 쉬운 프로젝트 고유 가드레일이 있다. Copilot 리뷰가 이 기준을 우선 확인하도록 저장소 지시사항을 짧고 강하게 작성한다.

## 범위

포함:

- `.github/copilot-instructions.md` 신규 추가
- Copilot Code Review가 우선 확인해야 할 저장소 코드 리뷰 지시사항 작성
- GitHub repository ruleset에 Copilot 자동 코드 리뷰 rule 추가
- 자동 리뷰 대상 브랜치, push 재리뷰, draft PR 리뷰 여부 설정
- 설정 결과와 검증 결과 문서화

제외:

- Copilot 요금제, 조직 권한, seat 관리 변경
- 기존 branch protection 또는 merge gate 강화
- 릴리스/서명/공증/Homebrew Cask 자동화 변경
- PR 템플릿 구조 변경
- Xcode project 직접 수정

## 설계 방향

Copilot 지시사항은 공식 지원 경로인 `.github/copilot-instructions.md`에 둔다. Code Review가 각 instruction 파일의 앞 4,000자만 읽는 제약을 고려해, 전체 저장소 규칙을 길게 복제하지 않고 리뷰에 필요한 핵심 위험만 압축한다.

자동 리뷰 ruleset은 `copilot_code_review` rule만 포함하는 독립 ruleset으로 만든다. 기존 `protect-main` ruleset은 비활성 상태이고 삭제/force-push 보호 성격이므로 이번 작업에서 수정하지 않는다.

대상 브랜치는 내부 task PR의 기본 대상인 `devel`을 우선 포함하고, release PR도 자동 리뷰를 받도록 `main`도 포함하는 방향을 검토한다. `review_on_push`는 켜서 PR 갱신마다 리뷰가 따라오게 하고, `review_draft_pull_requests`는 draft PR 노이즈와 quota 소모를 줄이기 위해 기본적으로 끈다.

- `test -f .github/copilot-instructions.md`
- `wc -c .github/copilot-instructions.md`
- `rg -n "RhwpCoreBridge|AppKit|FFI|project.yml|copilot_code_review|devel" .github/copilot-instructions.md mydocs/plans/task_m010_102_impl.md mydocs/working mydocs/report`
- `gh api repos/postmelee/alhangeul-macos/rulesets --jq '.[] | select(.name == "copilot-code-review")'`
- `gh api repos/postmelee/alhangeul-macos/rulesets/{ruleset_id}`
- `git diff --check`

Xcode build와 render smoke test는 이번 작업이 Swift/Rust 소스와 renderer 동작을 바꾸지 않으므로 기본 검증 범위에서 제외한다.

## 리스크

- Copilot 자동 리뷰는 PR 작성자 권한, Copilot Code Review 접근 권한, premium request quota에 영향을 받을 수 있다.
- GitHub Ruleset API의 rule schema가 변경되면 UI 확인 또는 API payload 보정이 필요할 수 있다.
- draft PR 리뷰를 켜면 하이퍼-워터폴 중간 산출물에 자동 리뷰가 과도하게 붙을 수 있고, quota 소모가 증가할 수 있다.
- `.github/copilot-instructions.md`가 길어지면 Code Review가 앞부분만 읽을 수 있으므로 향후 지시사항 증설 시 우선순위 관리가 필요하다.
