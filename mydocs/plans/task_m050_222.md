# Task M050 #222 수행계획서

## 목적

`samples/복학원서.hwp` Quick Look 미리보기에서 워터마크 이미지가 본문 텍스트와 표 위에 그려지는 문제를 해결한다.

완료 후 작업자는 Swift native renderer가 render tree의 `BehindText` 이미지 속성을 해석해, 문서 배경 위이면서 본문 텍스트와 표 아래인 레이어에 워터마크를 배치하는지 확인할 수 있어야 한다.

## 배경

현재 Quick Look preview는 `HwpPageImageRenderer`가 `RhwpDocument.renderPageTree` 결과를 받아 `CGTreeRenderer`로 CoreGraphics bitmap을 생성한다. 이 경로는 rhwp-studio/WASM 화면과 다르며, render tree의 노드 순서를 그대로 순회한다.

`samples/복학원서.hwp` 분석 결과 중앙 워터마크 이미지는 page-level `Image` 노드이고 JSON 속성에 `text_wrap: "BehindText"`가 포함되어 있다. 그러나 Swift `ImageNode`는 `text_wrap`을 디코딩하지 않고, `CGTreeRenderer`는 top-level 자식 순서대로 body 이후에 이미지를 그리므로 워터마크가 본문 위에 나타난다.

흑백/GrayScale 효과 미지원은 upstream `rhwp` 릴리즈 갱신으로 해결될 별도 문제로 보고, 본 작업에서는 레이어 순서 문제만 다룬다.

## 범위

포함:

- `RenderTree.ImageNode`에서 `text_wrap` 속성 디코딩
- `CGTreeRenderer`에서 `BehindText` 이미지의 draw order 조정
- `samples/복학원서.hwp` 1페이지 기준 워터마크가 본문 아래로 내려가는지 검증
- 기존 이미지, header/footer, page background 렌더링 순서 회귀 확인
- 필요한 경우 render debug helper에 레이어 진단 출력 보강

제외:

- upstream `rhwp` 갱신과 GrayScale/흑백 효과 반영
- PageLayerTree 전면 전환
- rhwp-studio/WASM asset 갱신
- PDF export semantics 변경
- Quick Look UI/chrome 변경

## 설계 방향

Swift native renderer는 render tree의 순서만 신뢰하지 않고, HWP draw order 의미를 반영해야 한다. 우선 1차 구현은 `ImageNode.textWrap`을 좁게 추가하고, 값이 `BehindText`인 page-level 이미지를 본문/표/일반 도형보다 먼저 그리는 방식으로 제한한다.

렌더 패스는 기존 노드 구조를 크게 바꾸지 않고 `CGTreeRenderer` 내부에서 분리한다. `PageBackground`는 가장 먼저 유지하고, `BehindText` 이미지는 그 다음, 일반 body/header/footer/foreground 노드는 기존 순서를 최대한 유지한다. 같은 문서의 좌상단 로고도 `BehindText`로 표시되므로, 단순히 모든 `BehindText`를 body 아래로 내렸을 때 header 영역 로고 위치가 깨지지 않는지 함께 확인한다.

## 리스크

- `text_wrap` 의미가 page-level 이미지 외 control/object에도 적용될 수 있어, 1차 구현 범위를 너무 넓히면 다른 문서의 z-order가 바뀔 수 있다.
- 좌상단 로고처럼 작은 `BehindText` 이미지도 함께 이동하므로 header/body와의 상대 순서를 주의해 검증해야 한다.
- upstream core SVG도 현재 동일하게 body 이후 이미지를 내보낼 수 있어, Swift 쪽 보정과 core 경로 전환 판단을 혼동하지 않아야 한다.
- GrayScale 효과가 아직 적용되지 않으면 수정 후에도 rhwp-studio 최신 화면과 완전한 색상 일치는 나지 않는다.
