## 설계 방향

이 저장소의 `Casks/alhangeul-macos.rb`를 release 기준 Cask source로 유지하고, 같은 내용을 `postmelee/homebrew-tap`에 반영한다. Homebrew 사용자 안내는 tap context 검증이 끝난 뒤에만 공개한다.

Cask는 단일 universal DMG URL과 고정 SHA256을 사용한다. `on_arm`/`on_intel` 분기는 추가하지 않는다.

1. Stage 1: Cask source와 문서 안내 갱신

   - 사용자-facing Homebrew 안내 문구를 tap 검증 후 공개 가능한 형태로 정리한다.
2. Stage 2: tap 반영과 Homebrew 검증

   - tap context에서 style/audit/install/uninstall smoke를 수행한다.
3. Stage 3: 최종 보고와 PR 게시
   - 검증 결과와 tap URL을 release 기록과 최종 보고서에 반영한다.
   - `publish/task209` PR을 게시한다.

## 검증 계획

```bash
./scripts/update-cask-sha256.sh --dry-run 0.1.2 <public-sha256-file>
brew style --cask alhangeul-macos
brew audit --cask alhangeul-macos
brew audit --cask --new alhangeul-macos
brew install --cask postmelee/tap/alhangeul-macos
brew uninstall --cask alhangeul-macos
git diff --check
```

`brew audit --cask --new`는 official Homebrew/homebrew-cask 신규 제출 수준의 참고 검증으로만 기록한다. maintainer tap 공개 gate는 일반 `brew style`, `brew audit`, install/uninstall smoke를 기준으로 한다.

## 리스크

- tap 저장소 push와 public 설치 안내는 외부 사용자에게 즉시 보이는 변경이다.
- local machine에 남은 LaunchServices/Quick Look 등록 cache가 install/uninstall smoke 결과를 흐릴 수 있다.
- `brew audit --new`는 token notability나 platform suffix 기준으로 실패할 수 있으며, maintainer tap 공개 차단 기준으로 취급하지 않는다.
