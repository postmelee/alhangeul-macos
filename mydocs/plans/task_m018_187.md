# Task M018 #187 수행계획서

## 목적

`v0.1.1` public release 전에 Homebrew tap 배포 경로와 Cask 검증 절차를 확정한다. GitHub Release DMG와 SHA256이 확정된 뒤 어떤 tap에 어떤 절차로 `Casks/alhangeul-macos.rb`를 반영하고 검증할지 문서와 릴리즈 안내에 일관되게 남긴다.

## 배경

`v0.1.0` release에서는 저장소 안의 `Casks/alhangeul-macos.rb`를 유지했지만 실제 Homebrew tap 공개는 수행하지 않았다. raw path 기준 `brew audit --cask Casks/alhangeul-macos.rb`가 Homebrew 환경에서 거부될 수 있으므로, v0.1.1에서 Homebrew 설치 경로를 공개하려면 tap context 기준 검증 방식과 공개 여부를 먼저 정해야 한다.

이 작업은 milestone 18 순서상 #188 public release 실행 전에 완료해야 한다. #208이 Intel Mac 지원과 단일 universal DMG 안내를 다루므로, 본 작업에서는 단일 universal DMG 정책과 Homebrew Cask URL/SHA256 기준이 충돌하지 않는지도 확인한다.

## 범위

포함:

- 후보 tap(`postmelee/homebrew-alhangeul`, `postmelee/homebrew-tap`, 장기 후보 `Homebrew/homebrew-cask`)의 운영 장단점 정리
- v0.1.1에서 Homebrew 설치 안내를 공개할지 여부와 조건 정리
- `Casks/alhangeul-macos.rb`를 tap context에 반영하는 절차 정리
- `brew style`, `brew audit`, 설치 검증 명령과 실패 시 기록 기준 정리
- release manual, release note template, v0.1.1 release record의 Homebrew 관련 문구 보정

제외:

- `Homebrew/homebrew-cask` 공식 제출
- `v0.1.1` public release workflow 실행
- Developer ID signing, notarization, app bundle 구조 변경
- 실제 public DMG SHA256 확정 후의 최종 Cask SHA 교체와 tap push. 필요 시 #188 또는 별도 승인 단계에서 수행한다.

## 설계 방향

초기 public patch release에서는 공식 Homebrew/homebrew-cask 제출보다 maintainer 소유 tap을 우선 검토한다. 저장소의 `Casks/alhangeul-macos.rb`는 release 기준 원본으로 유지하고, public DMG가 GitHub Release asset으로 올라간 뒤 선택한 tap에 복사하거나 동기화하는 흐름을 문서화한다.

검증은 raw path가 아니라 tap context를 기준으로 한다. public DMG가 아직 없거나 SHA256이 확정되지 않은 단계에서는 `sha256 :no_check` 또는 후보 checksum 상태를 public 배포 완료로 표현하지 않고, 최종 release checklist에서 확정 항목으로 남긴다.

## 잠정 단계

1. 현황 조사와 정책 결정안 작성
   - 현재 Cask source, release manual, release note template, v0.1.1 record를 대조한다.
   - 후보 tap별 운영 방식과 v0.1.1 공개 여부 결정안을 정리한다.
2. 문서와 release template 반영
   - 선택한 운영 기준을 release manual과 v0.1.1 record에 반영한다.
   - GitHub Release note template의 Homebrew 안내가 실제 공개 여부와 맞도록 보정한다.
3. 검증 절차 확인
   - 가능한 범위에서 `brew` 명령 interface와 Cask 문법 검증을 수행한다.
   - 네트워크나 tap 권한 때문에 실행하지 못하는 검증은 #188 handoff 항목으로 남긴다.
4. 단계 종료 보고
   - 변경 사항, 검증 결과, #188에서 이어서 수행할 공개/비공개 결정 항목을 보고한다.

## 검증 계획

- 문서 정합성:
  - `rg`로 Homebrew, Cask, DMG SHA256, release note 안내 문구를 대조한다.
  - `git diff --check`로 문서 공백과 패치 오류를 확인한다.
- 스크립트 검증:
  - `bash -n scripts/ci/write-release-notes.sh`
  - 가능하면 `scripts/ci/write-release-notes.sh` dry-run으로 Homebrew 안내 문구를 확인한다.
- Homebrew 검증:
  - `brew --version`
  - 가능하면 선택한 tap context 기준 `brew style --cask alhangeul-macos`
  - 가능하면 선택한 tap context 기준 `brew audit --cask --new alhangeul-macos`
  - public DMG가 아직 없으면 실제 `brew install --cask`는 #188 이후 검증 항목으로 남긴다.

## 리스크

- public DMG와 SHA256이 아직 확정되지 않은 상태에서 Cask를 공개 안내하면 사용자 설치 경로가 불안정해질 수 있다.
- `brew audit --cask --new`는 public tap 제출 수준의 엄격한 요구를 포함할 수 있어, maintainer tap 기준의 최소 검증과 구분이 필요하다.
- #208에서 단일 universal DMG 정책이 보정되면 Homebrew 안내의 지원 아키텍처 문구도 함께 맞춰야 한다.
- tap 생성이나 push 권한은 저장소 작업과 별도 권한이므로, 실제 외부 변경은 작업지시자 승인 후에만 수행해야 한다.

## 승인 요청 사항

- 본 작업은 maintainer 소유 tap 우선 검토, official Homebrew/homebrew-cask 제출 제외, public DMG SHA256 확정 전 tap push 제외를 전제로 진행해도 되는지 승인 요청한다.
- 수행계획서 승인 후 Stage 1 현황 조사와 정책 결정안 작성으로 진행한다.
