# Issue #40 수행 계획서

## 타스크

- GitHub Issue: #40
- 마일스톤: M050
- 제목: Dock/Spotlight 표시명 한영 현지화
- 작업 브랜치: `local/task40`
- 기준 브랜치: `origin/devel` `5a66f9b`

## 배경

Issue #33에서 Quick Look thumbnail smoke test 실패 원인은 Thumbnail provider 구현이 아니라 설치 app bundle filesystem path와 LaunchServices/PlugInKit 등록 상태라는 점을 확인했다.

그 결과 실제 설치 경로와 배포 zip 내부 `.app` 이름은 ASCII인 `AlhangeulMac.app`으로 유지하는 것이 안전하다고 정리했다. 다만 사용자가 Dock, Finder, Spotlight에서 보는 표시명은 한국어 환경에서는 `알한글`, 영어 환경에서는 `AlhangeulMac`으로 노출되는 것이 더 자연스럽다.

카카오톡 사례처럼 실제 파일명은 `KakaoTalk.app`으로 유지하면서 `ko.lproj/InfoPlist.strings`에서 `카카오톡` 표시명을 제공하는 방식이 이 요구에 맞다.

추가 확인 결과, 기본 `Info.plist`의 `CFBundleDisplayName`/`CFBundleName`을 한글로 직접 둔 상태에서는 실제 filesystem bundle name인 `AlhangeulMac.app`과 불일치한다. Apple의 bundle 표시명 규칙상 Finder/Spotlight가 이 불일치를 만나면 localized 표시명 대신 filesystem name을 우선할 수 있으므로, 기본 plist 값은 실제 bundle name과 맞는 ASCII 값으로 두고 한국어 표시는 `ko.lproj/InfoPlist.strings`에서 제공해야 한다.

## 목표

- 실제 filesystem app bundle name은 `AlhangeulMac.app`으로 유지한다.
- 한국어 환경 표시명은 `알한글`로 제공한다.
- 영어 환경 표시명은 `AlhangeulMac`으로 제공한다.
- Quick Look preview extension과 Thumbnail extension도 필요한 범위에서 `ko`/`en` 표시명을 제공한다.
- Issue #33에서 확보한 LaunchServices/PlugInKit 안정성 기준을 깨지 않는다.

## 비목표

- `AlhangeulMac.app` 자체를 `알한글.app`으로 rename하지 않는다.
- bundle identifier `com.postmelee.alhangeulmac` 계열은 변경하지 않는다.
- executable/module/product 내부 이름은 이번 작업에서 변경하지 않는다.
- 앱 아이콘, UI copy, localization infrastructure 전체 도입은 이번 범위에 포함하지 않는다.

## 조사 기준

- `Info.plist`의 `CFBundleDisplayName`, `CFBundleName`과 localized `InfoPlist.strings`의 우선순위를 확인한다.
- 기본 `Info.plist`의 `CFBundleDisplayName`, `CFBundleName`이 실제 `.app`/`.appex` filesystem name과 일치하는지 확인한다.
- `LSHasLocalizedDisplayName` 적용 여부를 확인한다.
- XcodeGen이 `*.lproj/InfoPlist.strings` 리소스를 각 target bundle에 포함하는 방식을 확인한다.
- app extension 표시명은 host app 표시명과 별개로 유지해야 하므로 extension별 리소스 포함 범위를 분리한다.

## 검증 계획

- `xcodegen generate`
- `plutil -lint`로 추가한 `InfoPlist.strings`와 기존 `Info.plist` 확인
- `xcodebuild -project AlhangeulMac.xcodeproj -scheme HostApp -configuration Debug -derivedDataPath build.noindex/DerivedData CODE_SIGNING_ALLOWED=NO build`
- build 산출물에 `ko.lproj/InfoPlist.strings`, `en.lproj/InfoPlist.strings`가 포함됐는지 확인
- 설치본 `/Users/melee/Applications/AlhangeulMac.app` 기준으로 LaunchServices/PlugInKit 등록 확인
- 가능한 CLI 범위에서 localized display name 확인
- 필요 시 Dock/Finder/Spotlight 표시명은 수동 확인 항목으로 보고서에 남긴다.
- `mdfind`로 `알한글`과 `AlhangeulMac` 검색 후보가 설치본을 찾는지 확인한다.
- Spotlight 전체 검색에 개발 build 산출물 `.app`이 설치본과 경쟁하지 않는지 확인한다.

## 리스크

- Spotlight/Dock 표시명은 LaunchServices와 Spotlight 캐시 영향을 받으므로 리소스 반영 후 즉시 갱신되지 않을 수 있다.
- `mdls`/`mdfind` 결과는 LaunchServices와 Spotlight cache 상태에 따라 늦게 갱신될 수 있으므로, plist 구조 검증과 설치본 재등록 결과를 함께 본다.
- extension bundle 표시명은 host app 표시명과 별개로 PlugInKit에 노출되므로 각 extension별 현지화 리소스 포함 여부를 따로 확인해야 한다.
- 일반 `build/DerivedData` 아래 생성된 Debug app이 Spotlight에 인덱싱되면 표준 설치본과 검색 결과에서 경쟁할 수 있다.
