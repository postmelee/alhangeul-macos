# Issue #52 수행 계획서

## 타스크

- GitHub Issue: #52
- 마일스톤: M010 (v0.1.0)
- 제목: 기존 PR 문서 링크를 merge 후 조회 가능한 고정 URL로 보정
- 작업 브랜치: `local/task52`
- 기준 브랜치: `origin/devel` `1f7c55a`

## 목적

기존에 merge된 PR 본문 중 문서 링크가 `publish/taskN` 브랜치나 PR 화면 상대 경로에 의존해 장기 탐색성이 떨어지는 항목을 확인하고, merge 이후에도 클릭 가능한 commit SHA 고정 GitHub blob URL로 보정한다. 보정 대상, 기준 SHA, 수정 전후 링크를 문서로 남겨 이후 PR 본문 작성 정책과 실제 과거 PR 상태가 서로 맞도록 한다.

## 배경

- PR #46, PR #50 등 일부 기존 PR 본문은 작업 당시의 게시 브랜치 또는 상대 경로 기준으로 문서 링크를 작성했다.
- `publish/taskN` 브랜치는 PR merge 후 삭제 대상이므로, 해당 브랜치에 의존한 링크는 장기 보존 링크로 적합하지 않다.
- GitHub PR 본문의 상대 링크는 PR 화면 경로 기준으로 해석될 수 있어, `mydocs/` 문서 탐색에 불편하거나 깨질 가능성이 있다.
- Task #49에서 향후 PR 작성 정책은 매뉴얼에 별도로 반영했으므로, 본 타스크는 기존 PR 본문 보정과 기록에 집중한다.

## 범위

### 포함

- PR #46 본문 문서 링크 상태 확인.
- PR #50 본문 문서 링크 상태 확인.
- 필요 시 동일 유형의 불안정 링크가 있는 기존 PR 추가 조사.
- `blob/publish/taskN/...` 또는 PR 상대 경로 문서 링크를 commit SHA 고정 `https://github.com/postmelee/alhangeul-macos/blob/{sha}/mydocs/...` 링크로 보정.
- 수정 대상 PR, 기준 SHA, 수정 전 링크, 수정 후 링크 기록 문서 작성.
- 변경 후 PR 본문 링크가 브라우저에서 접근 가능한 형태인지 확인.

### 제외

- 향후 PR 작성 정책의 추가 매뉴얼 변경. 해당 범위는 Task #49에서 다룬다.
- PR 본문 링크와 무관한 PR 설명, 제목, 라벨, 마일스톤 변경.
- 이슈 close, release, 배포, 서명, 공증 관련 작업.
- Swift, Rust, Xcode 프로젝트 소스 변경.

## 설계 방향

- merge된 PR 본문 보정은 PR별 실제 head SHA와 merge commit SHA를 먼저 확인한 뒤 진행한다.
- 문서가 PR head commit에 존재하는 경우 PR head SHA 고정 URL을 우선 사용한다. merge 이후의 통합 상태를 기준으로 삼는 것이 더 적합한 문서는 merge commit SHA 고정 URL을 사용한다.
- PR 본문 수정은 `gh pr edit` 또는 GitHub API로 수행하되, 수정 전 본문을 먼저 저장 가능한 형태로 확인해 변경 범위를 최소화한다.
- 작업 기록은 내부 타스크 문서로 남기며, PR 본문 전체를 복제하지 않고 링크 보정에 필요한 표 중심으로 작성한다.
- 현재 확인된 기준 정보는 다음과 같다.
  - PR #46: head `35ee83aea2a09e00a71218b66e25f88de804c6b6`, merge `e1e61ed5b99b9dd89043c6f47179d536635028b2`
  - PR #50: head `0df35a92fd476badb2ed27775a48d4ae0ef4ea1e`, merge `0a3443d975bd6f635cfe12ab86be356543fa6f13`

## 잠정 단계

1. **Stage 1 — 대상 PR 링크 조사**: PR #46, PR #50 본문과 문서 링크를 수집하고, 추가로 같은 유형의 불안정 링크가 있는 기존 PR을 제한적으로 확인한다.
2. **Stage 2 — 링크 기준 SHA 결정과 보정안 작성**: 각 링크별 PR head SHA 또는 merge commit SHA 기준을 정하고 수정 전후 URL 표를 작성한다.
3. **Stage 3 — PR 본문 보정과 접근 검증**: 승인된 보정안에 따라 PR 본문을 수정하고 고정 blob URL 접근성을 확인한다.
4. **Stage 4 — 기록 정리와 최종 보고**: 수정 대상, 기준 SHA, 검증 결과를 최종 보고서에 정리하고 오늘할일 상태를 갱신한다.

각 단계 완료 시 단계별 완료보고서(`mydocs/working/task_m010_52_stage{N}.md`)를 작성하고 승인 후 다음 단계로 진행한다.

## 검증 계획

- `gh pr view 46 --json body,headRefOid,mergeCommit` 및 PR #50 동일 명령으로 기준 SHA와 본문을 확인한다.
- 수정 대상 링크는 `https://github.com/postmelee/alhangeul-macos/blob/{sha}/mydocs/...` 형식인지 점검한다.
- 보정 후 PR 본문을 다시 조회해 상대 경로 또는 `blob/publish/taskN/` 문서 링크가 남아 있지 않은지 확인한다.
- 주요 고정 URL은 HTTP 접근 또는 GitHub CLI/브라우저 확인으로 404가 아닌지 검증한다.
- 문서 변경 후 `git diff --check`를 실행한다.

## 리스크

- merge commit에는 파일 경로나 내용이 PR head와 다르게 보일 수 있다. 문서가 실제로 존재하는 SHA를 우선 확인한 뒤 링크 기준을 결정한다.
- 이미 merge된 PR 본문 수정은 GitHub 원격 상태를 바꾸므로, 단계별 승인 전에는 수정하지 않는다.
- PR 본문에 여러 링크가 섞여 있을 경우 문서 링크만 보정하고 일반 이슈/PR 링크는 그대로 둬야 한다.
- GitHub API 권한 또는 네트워크 문제로 PR 본문 수정이 실패할 수 있다. 실패 시 수정 대상과 대체 실행 명령을 보고한다.
