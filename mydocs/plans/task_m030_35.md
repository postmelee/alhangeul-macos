# Task M030 #35 수행계획서

## 작업 개요

- 이슈: #35 Quick Look/Thumbnail 공통 group drawing 저해상도 렌더링 수정
- 마일스톤: `v0.3.0`
- 브랜치: `local/task35`
- 기준 브랜치: `devel`
- 시작 기준 커밋: `37dc664`
- 목표: `group-drawing-02.hwp`가 Quick Look preview와 Finder thumbnail에서 공통으로 낮은 해상도처럼 보이는 원인을 렌더링 경로 기준으로 분리하고, 필요한 수정 범위를 구현 계획서 단계로 확정한다.

## 배경

Issue #26에서 Finder thumbnail의 embedded preview 품질 게이트는 보완했지만, `group-drawing-02.hwp`는 full render fallback에서도 group drawing 영역이 작은 저해상도 이미지처럼 보이는 문제가 남았다. Finder thumbnail뿐 아니라 Quick Look preview에서도 같은 증상이 보이므로 `HwpPageImageRenderer`, `CGTreeRenderer`, render tree, 또는 core의 group drawing/image/shape 생성 경로를 함께 확인해야 한다.

## 작업 범위

- `group-drawing-02.hwp`의 render tree 구조 확인
- embedded preview 회피 이후 full render 경로가 실제로 사용되는지 재검증
- `Sources/Shared/HwpPageImageRenderer.swift`의 첫 페이지 bitmap 생성 경로 분석
- `Sources/RhwpCoreBridge`의 render tree 디코딩과 `CGTreeRenderer` group/image/shape 렌더링 경로 분석
- 필요 시 `Vendor/rhwp` core render tree 생성 경로를 읽기 전용으로 확인
- 수정이 필요한 경우 Swift bridge 또는 앱 저장소 소유 영역에 한정해 구현 계획서에서 단계화

## 제외 범위

- Dock/Spotlight 표시명, localized display name, packaging 정합화

- 편집 기능과 저장 기능
