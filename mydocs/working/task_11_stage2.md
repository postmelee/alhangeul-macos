# Issue #11 단계 2 완료 보고서

## 수행 내용

- `PR 처리 규칙`, `빌드 및 실행`, `rhwp Core Submodule 운영`, `Swift 및 macOS 코드 규칙`의 상세 절차를 `mydocs/manual/` 하위 분리 문서로 작성했다.
- `AGENTS.md`에는 각 항목별 강제 규칙만 남기고 manual 참조를 추가했다.
- `Git 워크플로우`는 작업 브랜치 생성/동기화/PR 대상 규칙을 항상 보이도록 `AGENTS.md` 본문에 유지했다.

## 판단

`AGENTS.md`는 에이전트가 매 작업 시작 시 읽는 문서이므로 길이를 줄이고 강제 가드레일을 강조하는 편이 실행 정확도를 높인다. 반면 상세 절차는 manual 문서로 분리해 필요 시 참조하게 하면 유지보수 비용을 낮출 수 있다.

## 결과

문서 구조를 `강제 규칙(AGENTS) + 상세 절차(manual)`로 일관화했고, 기존 요청대로 기준 브랜치/PR 방향성(`devel` 대상, upstream PR 금지)을 `AGENTS.md`에서 즉시 확인 가능하게 유지했다.
