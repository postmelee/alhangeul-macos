## 검증 결과와 #188 public smoke

현재까지 확인된 검증은 각 선행 task의 Debug/Release 또는 rehearsal 기준이다. #188에서 signed/notarized public DMG 설치본으로 반복해야 한다.

| 영역 | 현재 기준 | #188 반복 기준 |
|------|-----------|----------------|
| 창 확대/resize | #183 Debug build smoke 통과 | `v0.1.1` 설치본에서 `KTX.hwp`, `hwpx-01.hwpx` window zoom/resize 후 fallback 미표시 |
| Finder thumbnail | #199 Debug/Release 임시 등록본 thumbnail 생성 통과 | public DMG 설치본에서 HWP/HWPX `qlmanage -t -x`와 Finder icon view 확인 |
| DMG layout | #184 rehearsal DMG layout smoke 통과 | signed/notarized public DMG에서 root 구성, background, icon 위치, 첫 실행 안내 반복 확인 |
| Release notes | #185 dry-run template과 heading check 통과 | 실제 public DMG SHA256, GitHub Release body, Pages, appcast, release detail doc 대조 |
| Universal app/extension | #208 generic Release/package 산출물에서 app/extension 실행 파일 `arm64 + x86_64` 검증 통과 | public DMG 안의 `Alhangeul.app`, `AlhangeulPreview.appex`, `AlhangeulThumbnail.appex` 실행 파일 slice 재확인 |
| Legal notice | #215 Debug build에서 `NSHumanReadableCopyright`와 app bundle `Resources/Legal/*` 포함 및 사본 동일성 검증 통과 | signed/notarized public DMG를 mount한 뒤 `Alhangeul.app/Contents/Info.plist`와 `Contents/Resources/Legal/{LICENSE,THIRD_PARTY_LICENSES.md,FONTS.md}` 재확인 |
| Sparkle appcast / Pages deployment | script와 workflow 기준 존재. #206에서 generated appcast를 Pages artifact에 포함하고 `deploy-pages`로 배포하도록 전환 | Pages source `workflow`, `github-pages` `v*` tag policy, deploy-pages `page_url`, stable appcast item, EdDSA signature, release notes URL 확인 |

실행하지 않은 수동 확인 항목은 성공으로 기록하지 않는다. 실제 Intel Mac 실기기 실행 smoke는 접근 가능한 환경에서만 성공으로 기록하고, 미실행 시 #188 public smoke handoff에 사유를 남긴다. public DMG SHA256과 appcast EdDSA signature는 #188에서 확정하고, Cask SHA256과 Homebrew tap 검증 결과는 #209에서 확정한다.

## 알려진 제한 사항과 후속 이슈

- 앱 viewer/editor 화면은 bundled `rhwp-studio`를 WKWebView에서 실행한다.
- PDF 내보내기, Quick Look preview, Finder thumbnail은 Rust bridge와 Swift native renderer 계열 경로를 사용하므로 앱 화면과 표시가 다를 수 있다.
- 인쇄는 `rhwp-studio` page payload를 별도 WKWebView/PDFKit/AppKit 출력 경로로 처리한다.
- Quick Look/Thumbnail smoke 통과는 extension 등록과 기본 렌더 성공 확인이며, 모든 문서가 앱 화면과 같은 시각 결과로 보인다는 보장은 아니다.
- 손상, 대용량, 미지원 문서 fallback은 복구가 아니라 앱과 extension이 raw error, hang, crash로 끝나지 않게 하는 안전장치다.
- native renderer parity 개선은 Swift native viewer 방향에서 계속 다룬다.
- #198, #186, #187, #208, #206이 #188 전에 완료되어야 하며, #209는 #188에서 public DMG URL과 SHA256을 얻은 뒤 진행한다.
- Sparkle appcast와 Homebrew Cask는 아키텍처별 asset URL을 나누지 않고 같은 public universal DMG URL을 사용한다.
- Pages/appcast 배포는 `Release Publish DMG` workflow의 `deploy-pages` job 성공과 public `appcast.xml` URL 반영을 기준으로 판단한다.
