Stage 4 기준 original public build `2` release workflow가 성공했고, 이후 Stage 5 설치본 smoke에서 Quick Look/Thumbnail extension render crash가 확인됐다. Stage 9에서는 About window가 sandboxed app 내부 `pluginkit` 조회 실패를 `시스템 등록 확인 불가`로 노출하던 문제를 고치고 build `4`로 올렸다. 최종 public respin은 `Release Publish DMG` run `25645869039`로 성공했고, stable Sparkle appcast는 `sparkle:version=4`, `sparkle:shortVersionString=0.1.1`, minimum system `12.0`, EdDSA signature를 포함한다. 작업지시자는 build `4` 재설치 후 Quick Look preview와 Finder thumbnail 정상 동작을 확인했다. `v0.1.0`에서 Sparkle 업데이트한 사용자에게 남을 수 있는 Finder thumbnail cache refresh UX는 #225로 이관한다.

## 사용자용 요약

`v0.1.1`은 `v0.1.0` public release 이후 확인된 설치본 smoke 결함과 배포 안내를 보강하는 patch release다. 핵심 변경은 창 확대/resize 시 WebView 오류 fallback 수정, Finder thumbnail 생성 hang 수정, DMG 설치 창과 첫 실행 안내 개선, Intel Mac과 Apple Silicon Mac을 같은 단일 universal DMG로 지원하는 release gate와 communication template 보강이다.

첨부된 About window 스크린샷의 `시스템 등록 확인 불가` 표시는 실제 PlugInKit 미등록을 의미하지 않았다. 터미널 기준 두 extension은 등록되어 있었고 provider도 실행됐다. 그러나 DiagnosticReports에서 `AlhangeulPreview`와 `AlhangeulThumbnail`의 `EXC_BAD_ACCESS` 크래시가 확인됐다.

| 대상 | 확인된 실패 |
|------|-------------|
| `AlhangeulPreview` | `HwpPageImageRenderer.renderPage(...)`에서 `CGContextFillRect` 경유 `EXC_BAD_ACCESS` |
| `AlhangeulThumbnail` | `HwpPageImageRenderer.renderPage(...)` / `CGTreeRenderer.drawTextClusters(...)` 경유 `EXC_BAD_ACCESS` |

원인은 Swift 배열을 `CGContext(data: &pixels, ...)` backing store로 넘긴 bitmap memory ownership 문제다. Stage 6에서 `CGContext(data: nil, bytesPerRow: 0, ...)`로 전환하고 extension OSLog를 추가했다. 단일 페이지 PNG reply, 다중 페이지 PDF reply 정책은 유지한다. 수정본은 source-level build와 renderer smoke를 통과했고, Stage 7 local ad-hoc clean install에서 thumbnail 생성과 crash 부재를 확인했다. Stage 8에서 respin candidate build를 `3`으로 올리고 local Release package smoke를 통과했다. 아직 signed/notarized respin 설치본 smoke는 완료되지 않았다.

## 알려진 제한 사항과 후속 이슈

- 앱 viewer/editor 화면은 bundled `rhwp-studio`를 WKWebView에서 실행한다.
- PDF 내보내기, Quick Look preview, Finder thumbnail은 Rust bridge와 Swift native renderer 계열 경로를 사용하므로 앱 화면과 표시가 다를 수 있다.
- 인쇄는 `rhwp-studio` page payload를 별도 WKWebView/PDFKit/AppKit 출력 경로로 처리한다.
- Quick Look/Thumbnail smoke 통과는 extension 등록과 기본 렌더 성공 확인이며, 모든 문서가 앱 화면과 같은 시각 결과로 보인다는 보장은 아니다.
- 손상, 대용량, 미지원 문서 fallback은 복구가 아니라 앱과 extension이 raw error, hang, crash로 끝나지 않게 하는 안전장치다.
- native renderer parity 개선은 Swift native viewer 방향에서 계속 다룬다.
- #183, #199, #184, #185, #198, #186, #187, #208, #206, #212, #215는 #188 전에 완료됐다.
- #209는 #188에서 확정한 final build `4` public DMG URL과 SHA256을 기준으로 진행한다.
- #225는 `v0.1.0` 또는 기존 설치본에서 Sparkle 업데이트한 사용자의 Finder thumbnail cache refresh UX, About rhwp version 표시, rhwp `v0.7.11` 반영, `v0.1.2` 배포를 추적한다.
- Sparkle appcast와 Homebrew Cask는 아키텍처별 asset URL을 나누지 않고 같은 public universal DMG URL을 사용한다.
- Pages/appcast 배포는 `Release Publish DMG` workflow의 `deploy-pages` job 성공과 public `appcast.xml` URL 반영을 기준으로 판단하며, Stage 4 original release와 Stage 9 build `4` respin에서 통과했다.
