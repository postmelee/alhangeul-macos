FAQ

릴리즈 기준 기능과 문서 처리 방식, 렌더링 경로에 대해 자주 묻는 내용을 정리했습니다.

알한글은 무료인가요?

네. 알한글은 MIT License로 공개되는 오픈소스 macOS 앱입니다. GitHub에서 소스 코드와 릴리즈를 확인할 수 있습니다.

어떤 파일을 지원하나요?

.hwp, .hwpx 문서의 Quick Look 미리보기, Finder 썸네일, 앱에서 열기와 기본 편집 흐름을 지원합니다. 현재 릴리즈 기준으로 PDF 내보내기, macOS 공유, 인쇄하기까지 함께 제공합니다.

문서는 외부 서버로 업로드되나요?

아니요. 알한글은 문서를 외부 서비스에 업로드하지 않고 Mac 안에서 처리하는 흐름을 우선합니다. 앱 뷰어도 로컬 파일을 앱 안에서 WKWebView로 전달해 표시합니다.

HWP/HWPX 편집과 저장은 어디까지 가능한가요?

앱 화면은 edwardkim/rhwp의 rhwp-studio를 WKWebView로 사용합니다. 문서를 열어 내용을 확인하고, rhwp-studio가 제공하는 범위 안에서 텍스트 중심의 기본 편집을 할 수 있습니다. HWP 문서는 저장과 다른 이름으로 저장을 지원하고, HWPX 문서는 현재 직접 저장이 제한되어 HWP 내보내기 경로를 사용합니다.

다운로드 파일은 어디에서 받을 수 있나요?

상단 다운로드 버튼은 최신 공식 릴리즈의 DMG 파일을 바로 내려받도록 연결됩니다. 첫 공식 릴리즈가 게시되기 전에는 파일이 준비되지 않을 수 있으며, 이때는 GitHub Releases에서 배포 상태를 확인할 수 있습니다.

앱 업데이트는 어떻게 확인하나요?

알한글은 Sparkle appcast를 사용해 새 버전을 확인합니다. 앱 메뉴의 알한글 > 업데이트 확인...에서 수동으로 확인할 수 있고, appcast 고정 URL과 릴리즈 노트는 업데이트 안내 페이지에서 확인할 수 있습니다.

썸네일/Quick Look과 앱에서 보이는 화면이 다를 수 있나요?

가능합니다. Finder 썸네일과 Quick Look은 Swift native render 경로를 사용하고, 앱 뷰어와 편집 화면은 rhwp-studio WebView 경로를 사용합니다. 이 차이 때문에 일부 문서에서 글꼴, 배치, 페이지 표현이 다르게 보일 수 있습니다. 궁극적으로는 WebView 기능을 native로 전환하는 것이 목표이며, GitHub 기여를 환영합니다.

앱 편집 화면의 오류는 어디에 제보하면 되나요?

편집 화면은 rhwp-studio를 WebView로 사용하므로, 렌더링과 편집 엔진 자체의 오류는 rhwp upstream에 제보하는 것이 가장 정확합니다. 알한글은 upstream 업데이트가 반영되면 macOS 앱에 맞춰 가져옵니다.

알한글은 edwardkim/rhwp를 코어로 사용합니다. HWP/HWPX 문서를 웹과 네이티브 환경에서 다룰 수 있게 만든 이 프로젝트 덕분에 알한글도 탄생할 수 있었습니다.
